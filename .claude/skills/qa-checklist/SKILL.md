# QA 체크리스트 스킬

기획서·화면정의서 기반으로 기능별 테스트 시나리오를 도출하고, 검수 체크리스트를 생성한다.

## Workflow

### Phase 1: 기능·정책 추출
- 대상 기능의 기획서 정의 확인
- 정상 흐름(Happy Path) 시나리오 정리
- 관련 정책·규칙·제한 사항 목록화
- 연관 기능(다른 기능에 영향을 주고받는 것) 파악

### Phase 2: 테스트 케이스 설계
아래 유형별로 시나리오를 도출:

**정상 케이스**
- 기본 정상 흐름 (Happy Path)
- 다양한 정상 입력값 조합

**예외 케이스**
- 필수값 누락
- 잘못된 형식 입력
- 권한 없는 사용자 접근
- 비즈니스 규칙 위반

**경계값 케이스**
- 최소/최대값
- 글자수 제한
- 수량/금액 한도
- 0, 음수, 특수문자

**상태 전환 케이스**
- 각 상태에서 가능한 전환
- 불가능해야 하는 전환 시도
- 동시 접근/변경 상황

**환경 케이스 (해당 시)**
- PC / 모바일 웹 / 앱 웹뷰
- 주요 브라우저별
- 네트워크 불안정 상황

### Phase 3: 출력

## Output Format

### 검수 개요
> 대상 기능, 검수 범위, 전제 조건 요약

|-------|---------|------|---------|----------|-----------|----------|------|------|

## Constraints
- 개발 단위 테스트가 아닌 **기획 기준 인수 테스트** 관점
- 각 TC는 재현 가능할 만큼 구체적으로 작성
- 추측으로 Pass/Fail 판정하지 않는다 — 직접 확인한 것만 기록
- P0 이슈는 반드시 재현 경로를 포함
