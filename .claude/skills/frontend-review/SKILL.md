# 프론트엔드 리뷰 스킬

화면정의서·기획서 기준으로 프론트엔드 구현의 누락·차이·이슈를 식별한다.

## Workflow

### Phase 1: 기준 확인
- 기획서/화면정의서에서 해당 화면의 정의 내용 확인
- 정의된 상태 전환, 인터랙션, 조건부 노출 로직 목록화
- 예외 처리(빈 상태, 에러, 권한 없음 등) 정의 확인

### Phase 2: 구현 대조
아래 항목을 기획서 대비 점검:

**화면 요소**
- 정의된 UI 요소가 모두 존재하는가
- 텍스트·레이블이 기획서와 일치하는가
- 정렬, 순서가 기획서와 일치하는가

**인터랙션**
- 클릭/탭 시 정의된 동작이 수행되는가
- 상태 전환(활성/비활성, 펼침/접힘 등)이 기획과 일치하는가
- 입력 필드의 유효성 검사가 기획 정책과 일치하는가

**예외 처리**
- 빈 상태(Empty) 화면이 구현되었는가
- 에러 발생 시 정의된 메시지가 노출되는가
- 로딩 상태가 구현되었는가
- 네트워크 오류 시 처리가 있는가

**반응형/크로스브라우저**
- PC/모바일 뷰가 기획서 대응 범위와 일치하는가
- 주요 브라우저에서 레이아웃이 깨지지 않는가

### Phase 3: 출력

## Output Format

### 리뷰 요약
> 전체 구현 상태 1~2문장 요약

### 이슈 목록 (화면별)

#### [화면명]

| # | 항목 | 유형 | 심각도 | 기획서 정의 | 실제 구현 | 비고 |
|---|------|------|--------|-----------|----------|------|
| 1 |      | 누락/불일치/미구현 | Critical/Warning/Minor |  |  |  |

### 확인 필요 사항
- 기획서에도 정의가 불명확하여 확인이 필요한 항목

## Constraints
- 코드 품질이 아닌 **기획 대비 구현 정합성**에만 집중
- 기획서에 정의되지 않은 항목은 "기획 정의 필요"로 분류
- 추측으로 Pass/Fail 판정하지 않는다
