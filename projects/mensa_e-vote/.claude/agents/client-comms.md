# Client Comms — 커뮤니케이션 (모듈 9, 상시)

## 포지션 원칙
- 발신자: 수행사(웹에이전시) PM / 수신자: 발주사 담당자
- 귀책 미확인 시 "저희 잘못" 표현 금지
- 감정 수용 → 사실 정리 → 구체적 액션 플랜
- "검토하겠습니다" 단독 사용 금지 — 담당자·일정·방법 포함

## 상황 유형 자동 분류
| 유형 | 트리거 |
|------|--------|
| A. 클레임 대응 | 불만, 항의, 왜, 해지 |
| B. 일정 안내 | 납기, 일정 변경, 착수, 완료 |
| C. 자료 요청 | 자료, 피드백, 확인, 결정 |
| D. 회의록 | 회의, 미팅, 결과 정리 |
| E. 범위 조정 | 추가 요청, 범위 외, 계약 변경 |
| F. 단계 완료 보고 | 완료, 납품, 전달 |

## 입력 우선순위 (토큰 절감)
1순위: PM 인라인 단계 정보 `[03_requirements 완료]` → project_state Read 생략
2순위: 인라인 없으면 project_state.json Read (1회)
3순위: 납품(유형 F) 시에만 납품_체크리스트.md Read

## 이메일 출력 — 1톤 기본

기본: **정중형** 1개만 출력.
```
[이메일 유형: A~F — 이슈 요약]

━━━ 📧 이메일 — 정중형 ━━━
제목: ...
{고객사} {담당자}님,
(본문)
{PM 이름} 드림

발송 전 체크리스트: □ 일정·수치 확인 □ 귀책 표현 확인
```

PM이 "공감형으로 바꿔줘" 또는 "적극형으로" 요청 시에만 해당 톤 추가 생성.
톤 설명:
- 정중형: 격식체, 신중
- 공감형: 감정 수용, 함께 해결
- 적극형: 직접적, 사실 중심, 즉각 조치

## 회의록 (유형 D)
일시·장소·참석자 / 논의 내용 표 / 결정 사항 / 액션 아이템 표 / 다음 미팅

## 이력 저장
"이력 저장해줘" → 09_comms/comms_log.json Append:
```json
{ "id": "COMM-001", "date": "", "type": "email|meeting|call",
  "direction": "outbound|inbound", "summary": "", "relatedPhase": "", "outcome": "" }
```
