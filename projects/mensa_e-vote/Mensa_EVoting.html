1. 프로젝트 개요

1-1. 목적

총회 현장에서 안건별 단일선택 전자투표를 진행하고, 투표 종료 즉시 결과를 집계하여 관리자에게 제공하는 시스템을 구축한다. 시스템은 쇼핑몰 서버 내부에 구축하여 회원 DB 및 검표 데이터를 직접 조회한다.

1-2. MVP 범위

포함 (MVP) 제외 (v2 이후)
관리자: 총회 등록, 안건 CRUD, 투표 열기/닫기, 총회 상태 관리, 의장 일괄 투표
투표자: 본인 인증 → 안건 목록 → 투표
위임: 구글 스프레드시트 기반 위임장 연동, 의장(관리자) 자동 귀속
출석: 쇼핑몰 검표 DB 직접 조회 (동일 서버)
결과: 안건별 득표율, 참여자 명단(익명 투표 원칙 준수) 		복수선택 투표
자체 위임 등록 화면
투표자용 별도 결과 화면
카카오 알림톡 발송
다중 관리자 권한
의장 위임 건별 개별 선택 투표
감사 로그(audit log)
아카이브·과거 총회 조회

항목 값

2. 용어 정의

용어 정의
총회 (Assembly) 투표가 진행되는 단위 행사. 상태: 준비중 → 투표 진행중 → 종료
안건 (Agenda) 총회 내 개별 투표 항목. 상태: 닫힘(기본) ↔ 열림
보기 (Option) 안건 내 선택지. 개수 제한 없음.
정회원 회원 등급이 정회원 이상인 회원. 총회 투표권의 대상자.
회원번호 정회원 고유 식별자. 위임 매칭 및 투표권 귀속의 기준 키.
티켓구매자 총회 현장 참석을 위해 티켓을 구매한 정회원. 티켓 구매는 현장 출석의 전제 조건이며, 투표권 발생의 전제 조건은 아니다.
투표권 정회원 이상 회원 전원이 보유하는 안건별 1회의 의결 참여 권리. 행사 경로는 ① 현장 출석(티켓 구매 + 검표) 또는 ② 위임장 제출이다.
투표 행사권 투표권을 실제로 행사할 수 있는 자격.
① 본인 행사: 티켓 구매자 본인이 현장 검표(출석 확인)를 완료한 경우
② 위임 행사: 위임장을 통해 수임인에게 행사권을 이전한 경우 (위임인 출석 불요)
위임인 투표 행사권을 타인에게 위임하는 티켓구매자
수임인 위임을 받아 대신 투표를 행사하는 회원. 수임인 본인의 출석(검표)은 불요.
의장 관리자 = 의장. 별도 회원번호 없음. 위임장에서 수임인이 공란인 경우 의장에게 자동 귀속되며, 관리자 페이지에서 일괄 투표를 행사한다.
검표 현장 QR 스캔을 통한 출석 확인. 쇼핑몰 시스템에서 처리되며, 결과는 동일 DB에 기록된다.

2-1. 투표 행사권 발생 조건 상세

상황 투표권 투표 행사권 투표율 모수 포함 누가 투표하나
정회원 + 티켓 구매 + 검표 완료 + 위임 X 있음 본인에게 발생 포함 본인
정회원 + 티켓 미구매 + 위임 O (수임인 지정) 있음 수임인에게 이전 포함 수임인
정회원 + 티켓 미구매 + 위임 O (수임인 공란) 있음 의장(관리자)에게 귀속 포함 의장 (관리자 일괄 투표)
정회원 + 티켓 구매 + 위임 O (수임인 지정) + 투표 시작 전 검표 있음 위임 취소 → 본인 복귀 포함 본인
정회원 + 티켓 구매 + 위임 O + 투표 시작 후 검표 있음 위임 유지 포함 (수임인 기준) 수임인
정회원 + 티켓 미구매 + 위임 X + 불참 있음 행사 불가 제외 없음
정회원 + 티켓 구매 + 미검표 + 위임 X 있음 행사 불가 제외 없음

모수 기준 원칙: 투표 행사 가능 수(모수) = 출석 본인 + 유효 위임(수임인 지정 + 의장 귀속). 이 외의 모든 경우(불참·무위임, 티켓 구매했지만 미출석·무위임)는 모수에서 제외한다.

4-2. 데이터 연동

A. 쇼핑몰 DB 직접 조회

동일 서버에 구축하므로 외부 API 없이 DB 직접 조회로 처리한다.

항목 내용
조회 방식 동일 서버 DB 직접 SELECT (실시간)
회원 DB 조회 항목 회원번호, 이름, 전화번호
주문/티켓 DB 조회 항목 주문번호, 회원번호(FK), 검표 상태(Y/N), 검표 시각(timestamp)
현장 구매 대응 현장 판매분도 동일 DB에 즉시 반영되므로 별도 동기화 불필요

B. 위임 데이터 (구글 스프레드시트)

위임장은 외부 시스템에서 수합하여 구글 스프레드시트에 정리한다. 관리자가 동기화 버튼으로 Google Sheets API를 통해 투표 DB에 반영한다. 수임인 정보(이름, 전화번호, 회원번호)가 모두 공란인 경우, 의장(관리자)에게 자동 귀속된다.

항목 내용
시트명 "위임장"
시트 컬럼 A: 위임자명
B: 위임자휴대폰
C: 위임자회원번호
D: 수임자명
E: 수임자휴대폰
F: 수임자회원번호
매칭 기준 위임자: 회원번호(C열)로 회원 DB 매칭 (티켓 미구매 회원도 위임 가능)
수임자: 회원번호(F열)로 회원 DB 매칭
수임자 공란 처리 D·E·F 모두 공란 → 의장(관리자)에게 자동 귀속
동기화 방식 관리자 수동 동기화 버튼 (Google Sheets API v4)

스프레드시트 API 통신 관리

총회 상태 API 통신 비고
준비중 ✅ 통신 허용 동기화 버튼 활성화, 최신 시트 반영 가능
투표 진행중 ❌ 통신 차단 동기화 버튼 비활성화, API 호출 차단, 위임 데이터 동결
종료 ❌ 통신 차단 변경 불가

"투표 진행중" 전환 직전에 실행되는 강제 동기화가 위임 데이터의 최종 반영 시점이다. 전환 이후에는 스프레드시트가 변경되더라도 시스템에 반영되지 않으며, API 통신 자체가 차단된다.

5. 관리자 기능 상세

5-1. 로그인

항목 내용
인증 방식 ID / PW 별도 관리자 계정 (쇼핑몰 회원 계정과 분리)
역할 관리자 = 의장 (별도 회원번호 없음)
세션 유지 브라우저 세션 기반

5-2. 총회 등록·관리

항목 내용
총회명 필수 입력 (예: "2026년 정기총회")
일시 필수 입력
설명 선택 입력
상태 자동: 준비중 (등록 시 기본값)

의장은 관리자 본인이므로 별도 입력 필드가 없다. 위임장에서 수임인이 공란인 경우 자동으로 의장(관리자)에게 귀속되며, 관리자 페이지에서 일괄 투표를 행사한다.

5-3. 총회 상태 관리

상태 전환 조건 효과
준비중 총회 등록 시 기본값 안건 등록·수정·삭제 가능. 위임 스프레드시트 동기화 가능. 투표 불가.
투표 진행중 관리자가 수동 전환
(더블 컨펌 필수)		 ① 구글 스프레드시트 강제 동기화 1회 자동 실행
② 스프레드시트 API 통신 차단 (위임 데이터 동결)
③ 위임 취소 판정 일괄 실행 (원천 검표 시각 기준)
④ 투표 시작 시각(timestamp) 기록
⑤ 안건 열기/닫기 가능, 투표자 투표 가능
⑥ 의장 일괄 투표 가능
종료 관리자가 수동 전환 모든 안건 강제 닫힘. 결과 확정. 투표 불가.

"투표 진행중" 전환 시 더블 컨펌

단계 팝업 내용
1차 확인 "총회 상태를 '투표 진행중'으로 변경합니다. 계속하시겠습니까?"
2차 확인 "투표 진행으로 변경 시, 이후 위임인 본인이 출석하여도 수임인에게 투표권이 유지됩니다. 투표를 시작하시겠습니까?"

안건을 닫힘 → 열림으로 다시 열 수 있지만, 재오픈 시 "결과가 변동될 수 있습니다" 경고 팝업을 표시한다. 종료 상태에서는 안건 토글 불가하며, 종료는 되돌릴 수 없다. "투표 진행중 → 준비중" 롤백도 MVP에서는 지원하지 않는다.

5-4. 안건 관리

항목 내용
안건명 필수 입력
안건 설명 선택 입력
보기 무제한 추가 가능 (단일선택)
안건 번호 자동 부여 (제1호 안건, 제2호 안건, …)
상태 기본 닫힘, 관리자가 열림/닫힘 자유 토글

관리자 화면 상단에 전체 안건 수를 표시한다. (예: "전체 5건")

투표율 산식

구분 정의
투표 행사 가능 수 (모수) 출석 본인(위임 X) + 유효 위임(수임인 지정 + 의장 귀속) 건수. 불참·무위임자는 모수에서 제외.
투표 완료 수 해당 안건에 실제 투표를 제출한 수 (본인 + 위임 행사 + 의장 일괄 합산)
미투표 수 행사 가능 수 − 투표 완료 수
투표율 (투표 완료 수 / 투표 행사 가능 수) × 100

모수는 "투표 진행중" 전환 시점의 검표 현황 + 위임 판정 결과로 확정되며, 이후 변동하지 않는다. 관리자 화면에는 "전체 티켓 구매자 수"와 "투표 행사 가능 수"를 함께 표시한다.

관리자 결과 화면 통계 표시

표시 항목 설명
전체 정회원 수 참고 수치 (총회 투표권 대상자 전체)
전체 티켓 구매자 참고 수치 (현장 참석 신청자)
출석 본인 검표 완료 + 위임 없음 (또는 위임 취소 후 본인 복귀)
유효 위임 (수임인 지정) 수임인이 지정된 위임 건수
유효 위임 (의장 귀속) 수임인 공란 → 의장 귀속 건수
불참·무위임 모수에서 제외됨

수임인 유효성 검증

# 검증 항목 통과 조건 실패 시 처리
1 위임자가 회원 DB에 존재하는가 회원번호(C열)로 회원 DB 매칭 성공 + 정회원 이상 매칭 실패 상태 표기, 관리자 수동 확인
2 수임자가 회원 DB에 존재하는가 회원번호(F열)로 매칭 성공 매칭 실패 상태 표기, 관리자 수동 확인
3 수임자 공란 여부 D·E·F 모두 공란 의장(관리자)에게 자동 귀속
4 수임자 본인의 검표(출석) 여부 — 검표 불요 (수임인은 위임장만으로 투표 행사권을 가짐)
5 위임자와 수임자가 동일인 회원번호 기준 다른 사람 무효 처리, 관리자 알림
6 1인당 수임 한도 초과 수임 건수 ≤ 10 11건째부터 한도 초과 상태, 관리자 수동 판단

검증 결과는 관리자 위임 관리 화면에 상태 뱃지로 표시: 유효 매칭 실패 의장 귀속 무효 한도 초과

위임 취소 판정 로직

관리자가 총회 상태를 투표 진행중으로 전환하면(더블 컨펌 후), 시스템은 아래 절차를 자동 실행한다.

순서 동작
1 구글 스프레드시트 강제 동기화 1회 실행
2 스프레드시트 API 통신 차단 (이후 동기화 불가)
3 위임 목록 순회하며 위임인의 검표 상태 확인 (쇼핑몰 DB 직접 조회)
4 아래 조건에 따라 판정
5 판정 결과를 관리자 화면에 요약 표시
조건 처리
위임인의 검표 시각이 전환 시각 이전 위임 자동 취소 → 행사권 본인 복귀
위임인의 검표 시각이 전환 시각 이후 또는 미검표 위임 유지 → 수임인(또는 의장)이 행사

"투표 진행중" 전환 시 실행되는 강제 동기화가 위임 데이터의 최종 반영 시점이다. 전환 이후에는 스프레드시트가 변경되더라도 시스템에 반영되지 않으며, API 통신 자체가 차단된다. 이미 투표가 행사된 위임 관계는 어떤 경우에도 소급 변경하지 않는다. MVP에서는 총회 상태 롤백을 지원하지 않는다.

표시 항목 설명

유효 위임 수 (의장 귀속) 수임인 공란 → 의장 귀속 건수

5-7. 의장 일괄 투표

수임인이 지정되지 않아 의장(관리자)에게 귀속된 위임 건에 대해, 관리자 페이지에서 안건별로 일괄 투표를 행사한다.

항목 내용
대상 수임자 공란 → 의장 귀속된 위임 건 전체
투표 방식 안건별 보기 선택 → "일괄 투표" 버튼 → 귀속된 전체 건에 동일 보기 적용
투표 기록 DB에는 위임인 각각의 투표로 개별 INSERT. is_delegated = Y, delegated_to_chair = Y
참여자 명단 표시 위임인 이름으로 "투표함 / 위임 (의장 귀속)" 표시
수정 안건이 열림 상태인 동안 재선택 후 일괄 재적용 가능
건별 개별 선택 MVP에서 제외 — 전체 동일 보기 일괄 적용만 가능

의장 일괄 투표로 행사된 건은 해당 위임인 각각의 투표로 기록된다. 결과 집계 시 다른 투표와 동일하게 합산된다.
