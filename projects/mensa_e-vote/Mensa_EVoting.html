1. 프로젝트 개요

항목 내용
프로젝트명 정기총회 전자투표시스템
목적 정기총회 안건에 대해 현장 출석자 및 위임인의 전자 투표를 지원
대상 사용자 총회 관리자(의장), 현장 참석 정회원, 위임 수임인
예상 참여 인원 100~130명
투표 방식 안건별 단일 선택 (MVP)
개발 기간 10일 (4/16~4/25)
목표 오픈일 2026-04-25
구축 환경 쇼핑몰 서버 내 구축 (서버 분리 없음). 쇼핑몰 DB 직접 참조 전제로 기획.

2. 용어 정의

용어 정의
정회원 쇼핑몰 회원 등급이 정회원 이상인 회원. 총회 투표권의 대상자.
티켓구매자 총회 현장 참석을 위해 티켓을 구매한 정회원. 티켓 구매는 현장 출석의 전제 조건이며, 투표권 발생의 전제 조건은 아니다.
투표권 정회원 이상 회원 전원이 보유하는 안건별 1회의 의결 참여 권리. 행사 경로는 ① 현장 출석(티켓 구매 + 검표) 또는 ② 위임장 제출이다.
투표 행사권 투표권을 실제로 행사할 수 있는 상태. 출석(검표 완료) 시 본인에게, 위임 시 수임인에게 발생한다.
위임인 위임장을 제출하여 자신의 투표 행사권을 타인에게 이전하는 정회원. 티켓 미구매 회원도 가능.
수임인 위임장을 통해 타인의 투표 행사권을 위임받아 대리 투표하는 정회원.
의장 총회를 주재하는 관리자. 시스템에서는 관리자 계정 = 의장이다. 수임인이 지정되지 않은 위임건은 의장에게 자동 귀속된다.
총회 상태 준비중 → 투표 진행중 → 종료. 각 전환은 관리자가 수동으로 진행한다.
검표 현장에서 티켓 구매자의 출석을 확인하는 행위. 쇼핑몰 DB에 검표 시각(check_time)이 기록된다.

2-1. 투표 행사권 발생 조건 상세

상황 투표권 투표 행사권 투표율 모수 포함 누가 투표하나
정회원 + 티켓 구매 + 검표 완료 + 위임 X 있음 본인에게 발생 포함 본인
정회원 + 티켓 미구매 + 위임 O (수임인 지정) 있음 수임인에게 이전 포함 수임인
정회원 + 티켓 미구매 + 위임 O (수임인 공란) 있음 의장(관리자)에게 귀속 포함 의장 (관리자 일괄 투표)
정회원 + 티켓 구매 + 미검표 + 위임 O (수임인 지정) 있음 수임인에게 이전 포함 수임인
정회원 + 티켓 구매 + 미검표 + 위임 O (수임인 공란) 있음 의장(관리자)에게 귀속 포함 의장 (관리자 일괄 투표)
정회원 + 위임 O + "투표 진행중" 전환 시점에 검표 완료 상태 있음 위임 자동 취소 → 본인 복귀 포함 본인
정회원 + 위임 O + "투표 진행중" 전환 이후 검표 있음 위임 유지 포함 (수임인 기준) 수임인
정회원 + 티켓 미구매 + 위임 X + 불참 있음 행사 불가 제외 없음
정회원 + 티켓 구매 + 미검표 + 위임 X 있음 행사 불가 제외 없음

모수 기준 원칙: 투표 행사 가능 수(모수) = 출석 본인(검표 완료 + 위임 X) + 유효 위임(수임인 지정 + 의장 귀속). 이 외의 모든 경우는 모수에서 제외한다.

운영 권장: "투표 진행중" 전환 시점 이전에 충분한 입장·검표 시간을 확보해 현장 참석자가 최대한 본인 투표를 할 수 있도록 한다.

4. 시스템 흐름 · 데이터 연동

4-1. 전체 흐름

단계 관리자 행위 시스템 동작
1. 총회 생성 총회명, 일시 입력 Assembly 레코드 생성, 상태 = 준비중
2. 안건 등록 안건명 + 보기(선택지) 입력 Agenda + AgendaOption 레코드 생성
3. 위임장 동기화 동기화 버튼 클릭 구글 스프레드시트 API → Delegation 테이블 UPSERT
4. 출석 현황 확인 출석 관리 화면 조회 쇼핑몰 DB 직접 참조 (티켓 + 검표 시각)
5. 투표 시작 "투표 진행중" 전환 (더블 컨펌) ① 강제 동기화 1회 실행 → ② 검표 시각 기준 위임 취소 판정 → ③ 모수 확정 → ④ 스프레드시트 연동 차단
6. 안건 열기 안건별 "열기" 클릭 해당 안건 투표 가능 상태로 전환
7. 투표 진행 현황 모니터링 투표자 화면에서 선택지 제출 → Vote 레코드 생성 (안건 열림 중 수정 가능)
8. 안건 닫기 안건별 "닫기" 클릭 (더블 컨펌) 해당 안건 투표 마감, 결과 집계
9. 총회 종료 "종료" 클릭 (더블 컨펌) 전체 투표 마감, 최종 결과 확정. 되돌릴 수 없음.

투표자 인증 → 투표 흐름: 이름+전화번호+주문번호 뒤 5자리 입력 → 티켓 DB 매칭 → 성공 시 회원번호를 세션에 저장 → Delegation 테이블에서 delegatee_member_id로 위임 목록 조회 → 위임건이 있으면 수임인 드롭다운 구성 → 안건별 투표 진행.

4-2. 데이터 연동 구조

4-2-A. 출석 관리 (쇼핑몰 DB 직접 참조)

항목 내용
연동 방식 같은 서버 내 쇼핑몰 DB 직접 쿼리 (별도 API 불필요)
조회 대상 티켓 구매 테이블 + 검표 시각(check_time) 컬럼
데이터 갱신 관리자 화면에서 실시간 조회 (페이지 로드 또는 새로고침 시 DB 재조회)
표시 항목 구매자명, 전화번호, 주문번호, 검표 여부, 검표 시각

4-2-B. 위임장 관리 (구글 스프레드시트 연동)

항목 내용
연동 방식 Google Sheets API v4 + 서비스 계정 인증 (서버 사이드)
스프레드시트 컬럼 A: 위임자명 / B: 위임자휴대폰 / C: 위임자회원번호
D: 수임자명 / E: 수임자휴대폰 / F: 수임자회원번호
매칭 기준 위임자: 회원번호(C열)로 회원 DB 매칭 (티켓 미구매 회원도 위임 가능)
수임자: 회원번호(F열)로 회원 DB 매칭
동기화 방식 준비중: 관리자가 수동으로 동기화 버튼 클릭 (반복 가능)
투표 진행중 전환 시: 강제 동기화 1회 자동 실행 → 이후 연동 차단
투표 진행중 / 종료: 스프레드시트 API 통신 차단, 동기화 버튼 비활성화
수임인 공란 처리 수임인 정보(D·E·F열)가 모두 공란이면 의장(관리자)에게 자동 귀속
보안 서비스 계정 JSON 키는 서버 환경변수에 보관. 프론트엔드 노출 금지.

5. 관리자 기능 상세

5-1. 총회 관리

기능 설명
총회 생성 총회명, 일시 입력. 생성 시 상태 = "준비중".
총회 목록 생성된 총회 목록 표시. 상태별 필터 가능.

5-2. 안건 관리

기능 설명
안건 등록 안건명 입력 + 보기(선택지) 최소 2개 등록. "준비중" 및 "투표 진행중" 상태에서 가능. "종료" 후 불가.
안건 수정·삭제 "준비중" 상태에서만 가능. 투표 진행중 이후 수정·삭제 불가.
안건 열기·닫기 "투표 진행중" 상태에서 안건별 개별 토글. 열린 안건만 투표 가능.
행위 준비중 투표 진행중 종료
안건 추가 가능 가능 불가
안건 수정 가능 불가 불가
안건 삭제 가능 불가 불가
안건 열기·닫기 불가 가능 불가

긴급 안건 참고: 투표 진행중에 추가된 안건은 기존 확정된 모수(투표 행사 가능 수)를 동일하게 적용한다. 추가 후 관리자가 "열기"를 해야 투표가 시작되므로 기존 진행 중인 안건에 영향을 주지 않는다.

보기(선택지) 수정 규칙: 투표 진행중에 추가된 안건은 "열기" 전까지 보기 수정이 가능하다. 한 번 "열림" 상태가 되면 보기 수정·삭제 불가.

5-3. 총회 상태 관리

전환 조건 시스템 동작
준비중 → 투표 진행중 더블 컨펌 필수:
1차: "총회 상태를 '투표 진행중'으로 변경합니다. 계속하시겠습니까?"
2차: "투표가 시작되면, 이후 출석한 위임인의 투표권은 수임인에게 유지됩니다. 투표를 시작하시겠습니까?"		 ① 위임장 스프레드시트 강제 동기화 1회 실행
② 검표 시각 기준 위임 취소 판정
③ 투표 행사 가능 수(모수) 확정
④ 스프레드시트 API 통신 차단
투표 진행중 → 종료 더블 컨펌 필수:
1차: "총회를 종료합니다. 계속하시겠습니까?"
2차: "종료 후에는 되돌릴 수 없습니다. 모든 안건이 마감되고 최종 결과가 확정됩니다. 정말 종료하시겠습니까?"		 모든 안건 강제 닫힘. 최종 결과 확정. 되돌릴 수 없음.

롤백 불가: "투표 진행중 → 준비중" 롤백은 MVP에서 지원하지 않는다. "종료" 상태도 되돌릴 수 없다.

안건 가드: 등록된 안건이 0건인 상태에서는 "투표 진행중"으로 전환할 수 없다. 전환 시도 시 "안건을 1개 이상 등록하세요." 알림을 표시하고 전환을 차단한다.

안건 닫기 정책: 안건을 닫을 때 더블 컨펌을 수행한다.
1차: "해당 안건의 투표를 마감합니다. 닫으시겠습니까?"
2차: "닫힌 안건은 재오픈할 수 있지만, 재오픈 시 미투표자의 추가 투표로 결과가 변동될 수 있습니다. 정말 닫으시겠습니까?"

안건 재오픈 정책: 닫힌 안건을 다시 열 경우, 기존 투표 데이터는 유지한다. 아직 투표하지 않은 행사 가능자만 추가로 투표할 수 있다. 재오픈 시 경고 팝업: "기존 투표 결과에 미투표자의 투표가 추가되어 최종 결과가 변동될 수 있습니다. 재오픈하시겠습니까?"

횟수 제한: MVP에서는 재오픈 횟수에 제한을 두지 않는다. 단, 안건의 열기·닫기 이력(시각, 행위자)을 로그로 기록한다.

5-4. 투표 현황

표시 항목 설명
전체 정회원 수 총회 대상 정회원 이상 회원 수 (회원 DB 기준). 쿼리 불가 시 "미제공" 표시.
티켓 구매자 수 현장 참석 티켓 구매 회원 수
출석 본인 수 검표 완료 + 위임 X 인원
유효 위임 수 수임인 지정 위임 + 의장 귀속 위임 합산
불참·무위임 수 참고용 표시 (모수에서 제외)
투표 행사 가능 수 (모수) 출석 본인 + 유효 위임 (= 투표율 분모)
투표 완료 수 본인 투표 + 수임인 투표 + 의장 일괄 투표 합산
미투표 수 투표 행사 가능 수 − 투표 완료 수
투표율 투표 완료 수 / 투표 행사 가능 수 × 100

투표율 산식: 투표율(%) = (투표 완료 수 / 투표 행사 가능 수) × 100. 모수는 "투표 진행중" 전환 시점의 출석·위임 현황으로 고정한다.

정족수 판정: MVP에서는 정족수(의결 성립 요건) 판정을 포함하지 않는다. 시스템은 투표 결과(득표수·득표율·투표율)만 표시하며, 정족수 충족 여부는 의장이 별도로 판단한다.

익명성 원칙: 안건별로 전체 행사 가능자 수, 투표 완료 수, 미투표 수를 표시한다. 개별 회원이 어떤 보기를 선택했는지는 관리자 화면을 포함한 모든 UI에서 표시하지 않는다. 투표/미투표 상태만 목록으로 확인 가능하다. 10초 폴링으로 자동 갱신.

5-5. 결과 조회

항목 설명
표시 시점 안건이 closed 상태로 전환되면 관리자 화면에서 즉시 결과를 확인할 수 있다.
표시 항목 안건별로 다음 항목을 표시한다:
· 보기별 득표수
· 보기별 득표율 (해당 보기 득표수 ÷ 총 투표 수 × 100)
· 총 투표 수 (해당 안건에 투표한 총 인원)
· 모수 대비 투표율 (총 투표 수 ÷ eligible_count × 100)
정족수 판정 MVP에서는 미포함. 숫자만 표시하며, 가결/부결 판정은 의장이 별도 수행.
투표자 화면 MVP에서는 투표자 화면에 결과를 표시하지 않는다. 투표자는 자신의 투표 완료 여부와 선택한 보기만 확인 가능.

5-6. 출석 관리

기능 설명
출석 현황 조회 쇼핑몰 DB에서 티켓 구매자 + 검표 시각을 직접 조회하여 표시
표시 정보 구매자명, 전화번호, 주문번호, 검표 여부(Y/N), 검표 시각
위임 여부 표시 각 출석자 옆에 위임 상태 표시 (위임 없음 / 위임 있음 → 취소됨 / 위임 있음 → 유지)

5-7. 위임장 관리

5-7-A. 위임 검증 규칙

스프레드시트 동기화 시 아래 검증을 수행한다.

# 검증 항목 조건 실패 시 처리
1 위임자 자격 검증 회원번호(C열)로 회원 DB 매칭 + 총회 당일 기준 정회원 이상 매칭 실패 또는 위임자 자격 미달 표시, 관리자 수동 확인
2 수임인 공란 → 의장 귀속 D·E·F열 모두 공란 의장(관리자)에게 자동 귀속 처리
3 위임인 출석 여부 위임에는 출석(검표)이 불필요 -
4 위임인 = 수임인 동일인 C열 회원번호 = F열 회원번호 무효(동일인) 상태 표기
5 1인당 수임 건수 제한 제한 없음. 수임 건수에 상한을 두지 않는다. 관리자 화면에서 수임인별 건수를 확인할 수 있도록 카운터 표시
6 동일 위임자 중복 등록 동일 위임자 회원번호(C열)가 스프레드시트에 2행 이상 존재 마지막 행만 유효 처리. 관리자 화면에 중복 위임 경고 표시 + 이전 행은 무효(중복) 상태로 표기
7 수임인 자격 검증 회원번호(F열)로 회원 DB 매칭 + 총회 당일 기준 정회원 이상 수임인 자격 미달 표시, 관리자 수동 확인. 해당 위임건은 유효하지 않은 상태로 보류.

5-7-B. 위임 취소 판정 (투표 시작 시)

조건 처리
"투표 진행중" 전환 시점에 위임인의 검표 시각(check_time)이 존재하고, 검표 시각이 전환 시각보다 이전인 경우 위임 자동 취소 (cancelled_reason: 'delegator_attended') → 위임인 본인에게 투표 행사권 귀속
"투표 진행중" 전환 이후에 위임인이 검표 MVP에서는 위임 유지 → 수임인이 계속 투표. 위임인이 현장에 도착했더라도 전환 이후 검표이면 위임 취소하지 않음.

판정 기준: 동기화 반영 시각이 아니라 쇼핑몰 DB의 원천 검표 시각(check_time)을 기준으로 판단한다. "투표 진행중" 전환 버튼 클릭 시 즉시 동기화 1회 강제 실행 후, check_time < 전환 시각인 위임건을 자동 취소한다.

판정 시점: "투표 진행중" 전환 시 강제 동기화 직후, 검표 시각과 전환 시각을 비교하여 일괄 판정한다.

5-7-C. 위임 데이터 반영 컷오프

총회 상태 스프레드시트 동기화 위임 데이터 변경
준비중 가능 (수동 반복) 반영됨
투표 진행중 (전환 시) 강제 동기화 1회 자동 실행 이 시점이 최종 반영
투표 진행중 (전환 후) 차단 (API 통신 끊김, 버튼 비활성화) 변경 불가
종료 차단 변경 불가

5-7-D. 스프레드시트 동기화 실패 처리

상태 유지 원칙: 전환 시도 중 동기화가 실패하면 총회 상태는 "준비중"을 유지한다. 따라서 CSV 업로드를 포함한 준비중 단계의 모든 기능을 계속 사용할 수 있다.

기본 동작: "투표 진행중" 전환 시 강제 동기화가 실패하면, 자동으로 최대 3회까지 재시도한다. 3회 모두 실패할 경우 전환을 차단하고 "위임장 동기화에 3회 실패했습니다. CSV 업로드를 이용해주세요." 알림과 함께 CSV 업로드 버튼을 표시한다.

비상 전환: CSV 업로드도 불가능한 상황일 경우 관리자에게 추가 선택지를 제공한다.
· 마지막 성공 동기화 데이터가 있는 경우:
"동기화에 계속 실패하고 있습니다. 마지막으로 성공한 위임 데이터(YYYY-MM-DD HH:MM 기준)로 투표를 시작하시겠습니까?"
· 한 번도 동기화에 성공한 적이 없는 경우:
"위임 데이터 없이 투표를 시작하시겠습니까? 현장 출석자만 투표 가능합니다."

5-7-E. CSV 업로드 (API 장애 시 백업)

항목 설명
용도 구글 스프레드시트 API 연동 실패 시 백업 경로
절차 관리자가 구글 스프레드시트에서 CSV 다운로드 → 위임장 관리 화면에서 CSV 파일 업로드
파일 형식 UTF-8 CSV. 컬럼 순서는 스프레드시트와 동일 (A~F: 위임자명, 위임자휴대폰, 위임자회원번호, 수임자명, 수임자휴대폰, 수임자회원번호)
검증 API 동기화와 동일한 7개 검증 규칙 적용
처리 기존 위임 데이터를 전체 교체 (UPSERT). 업로드 시각이 마지막 동기화 시각으로 기록됨.
사용 가능 상태 "준비중"에서만 사용 가능. "투표 진행중" 전환 후에는 CSV 업로드도 차단.

CSV 업로드 오류 처리:
· 파일 인코딩이 UTF-8이 아닌 경우: "업로드 실패: 파일 인코딩을 확인해주세요. UTF-8 형식만 지원합니다."
· 컬럼 수가 6개가 아닌 경우: "업로드 실패: 파일 형식을 확인해주세요. UTF-8 CSV, 6개 컬럼 필수."
· 빈 파일(데이터 행 0건): "업로드 실패: 데이터가 없습니다. 파일 내용을 확인해주세요."
· 업로드 성공 후 검증 실패 행이 존재하면, 정상 행은 반영하고 실패 행은 목록으로 표시한다.

5-8. 의장 귀속 위임 일괄 투표

실행 조건: 총회 상태가 "투표 진행중"이고, 해당 안건이 "open(열림)" 상태일 때만 의장 일괄 투표가 가능하다. 안건이 닫혀 있거나 준비중이면 일괄 투표 버튼은 비활성화된다.

의장 일괄 투표 수정: 의장 일괄 투표도 동일하게 안건이 open 상태인 동안 수정 가능하다. 안건이 closed 상태로 전환되면 수정 불가. 재오픈 시 미투표 귀속건만 추가 투표 가능하며, 이미 투표된 귀속건은 수정할 수 있다.

항목 설명
대상 수임인이 지정되지 않아 의장(관리자)에게 귀속된 위임건
투표 방식 관리자 화면에서 안건별로 보기를 선택 → "일괄 투표" 버튼 클릭
일괄 처리 선택한 보기가 의장 귀속 전체 위임건에 동일하게 적용됨 (건별 개별 선택은 v2)
DB 기록 각 위임인의 회원번호(voter_member_id)별로 개별 Vote 레코드 생성. voted_by = 관리자 계정 ID.
참석자 목록 표시 해당 위임인 이름 옆에 투표함 (위임 — 의장 귀속) 표시

6. 투표자 기능 상세

6-1. 본인 인증

항목 설명
인증 방식 이름 + 전화번호 + 주문번호 뒤 5자리 입력 (별도 인증, 로그인 세션 미사용)
검증 쇼핑몰 DB의 티켓 구매 테이블에서 3개 값 일치 여부 확인
실패 시 "일치하는 정보가 없습니다. 입력 내용을 확인해주세요." 메시지 표시
인증 후 처리 인증 성공 시 해당 주문의 회원번호를 세션에 저장한다. 이후 위임 매칭·투표 기록에 회원번호를 사용한다.
세션의 회원번호로 Delegation 테이블(delegatee_member_id)을 조회하여 위임받은 건이 있으면 수임인 드롭다운을 자동 구성한다.
세션 유지 인증 성공 시 회원번호를 세션에 저장한다. 세션은 해당 총회가 "종료" 상태로 전환될 때까지 유지된다.
총회 종료 시 모든 투표자 세션이 만료되며, 이후 투표 페이지 접근 시 "총회가 종료되었습니다." 안내를 표시한다.
브라우저를 닫았다가 재접속한 경우, 세션이 유효하면 재인증 없이 기존 투표 상태가 복원된다.
재인증 시 복원 브라우저 새로고침·세션 만료 후 재인증 시, DB를 조회하여 기투표 안건은 "투표 완료" 상태로 복원한다.

향후 개선 고려: 쇼핑몰 서버 내 구축 환경이므로, 쇼핑몰 로그인 세션을 활용하면 별도 입력 없이 회원번호로 자동 식별이 가능하다. 100~130명이 현장에서 모바일로 동시 접속하는 상황에서 입력 단계를 줄이면 UX 및 운영 효율이 향상될 수 있다. MVP에서는 별도 인증 방식을 유지하되, 이후 로그인 세션 기반 인증으로 전환을 검토한다.

6-2. 투표 화면

항목 설명
안건 목록 모든 안건을 상태와 함께 표시한다.
· 준비중(draft): "예정" 뱃지 표시. 안건명·보기 확인 가능. 투표 비활성화.
· 열림(open): 보기(선택지) 라디오 버튼 활성화. 투표 가능.
· 닫힘(closed): "마감" 뱃지 표시. 라디오 버튼·투표 버튼 비활성화.
draft 안건 안내 투표자 화면에서 draft(준비중) 안건은 "예정" 뱃지와 함께 표시되며, 하단에 "안건 준비 중 — 보기(선택지)가 변경될 수 있습니다." 안내 문구를 표시한다.
투표 제출 선택 후 "투표" 버튼 클릭 → 확인 팝업: "투표를 제출합니다. 안건이 열려 있는 동안 수정할 수 있습니다." → 제출.
제출 후 해당 안건이 open 상태이면 "수정" 버튼이 표시되며, 선택을 변경하고 재제출할 수 있다.
안건이 closed 상태로 전환되면 수정 불가.
투표 수정 안건이 "열림" 상태인 동안 투표 수정이 자유롭게 가능하다. 제출 완료 후 "수정" 버튼이 표시되며, 클릭 시 선택을 변경하고 재제출할 수 있다. 안건이 닫히면 수정 불가.
투표 완료 표시 투표한 안건은 "투표 완료" 뱃지 + 선택한 보기 표시 + "수정" 버튼 (안건 열림 시에만 활성화)
투표 제출 실패 네트워크 오류·DB 타임아웃 등으로 제출 실패 시 "투표 저장에 실패했습니다. 다시 시도해주세요." 알림 표시. UNIQUE 제약 조건으로 중복 제출은 자동 방지.
안건 상태 관리자 화면 투표자 화면
draft (준비중) 표시 + 수정·삭제 가능 표시 + "예정" 뱃지 + 투표 비활성화 + "보기 변경 가능" 안내
open (열림) 표시 + 닫기 가능 표시 + 투표 가능 + 수정 가능
closed (닫힘) 표시 + 재오픈 가능 + 결과 즉시 표시 표시 + "마감" 뱃지 + 투표·수정 비활성화 + 결과 미표시

브라우저 새로고침·이탈 안내: 투표를 제출하기 전에 브라우저를 새로고침하거나 페이지를 이탈하면, 선택 중이던 내용은 보존되지 않는다. 이미 제출 완료된 투표는 DB에서 조회하여 자동 복원된다. 투표 화면 상단에 "투표는 제출 후에만 저장됩니다. 새로고침 시 미제출 선택은 초기화됩니다." 안내 문구를 상시 표시한다.

6-2-A. 수임인 투표 화면 구조

수임인이 위임을 1건 이상 받은 경우, 투표 화면 상단에 드롭다운 선택기가 표시된다.

항목 설명
레이아웃 전체 진행 요약 + 드롭다운 선택기 영역은 화면 상단 고정(sticky) 처리. 안건 목록을 스크롤해도 항상 보이도록 한다.
전체 진행 요약 드롭다운 상단에 전체 진행 현황을 요약 표시.
예: 전체 진행: 8/15 완료 (3명 중 1명 미완료)
드롭다운 구성 첫 번째 항목: 내 투표 (홍길동) — 3/5 완료 — 본인 명의
이후 항목: 위임 투표 — 김위임 (12345) — 0/5 미완료 형식으로 위임인별 표시
미완료 명의가 상단으로 자동 정렬, 완료 명의는 하단으로 이동
동작 방식 드롭다운에서 선택 → 해당 명의의 안건 목록·투표 상태가 화면에 표시됨. 각 명의별로 독립 투표.
투표 완료 표시 미완료: 위임 투표 — 김위임 (12345) — 2/5 미완료
완료: 위임 투표 — 박대리 (67890) — 5/5 완료 ✓
미완료 이탈 경고 투표자가 페이지를 이탈하려 할 때 미완료 명의가 있으면
"위임 투표 — 김위임 외 N건이 미완료입니다. 페이지를 나가시겠습니까?" 알림 표시
제한 사항 의장 귀속 위임건은 수임인 드롭다운에 표시되지 않음 (관리자 화면에서만 일괄 처리)

7. 데이터 모델

쇼핑몰 DB 내에 아래 테이블을 추가한다. 기존 쇼핑몰 테이블(회원, 주문/티켓)은 참조만 한다.

7-1. Assembly (총회)

컬럼 타입 설명
id INT, PK 총회 고유 ID
title VARCHAR 총회명
scheduled_at DATETIME 총회 예정 일시
status ENUM 'preparing' / 'voting' / 'closed'
voting_started_at DATETIME "투표 진행중" 전환 시각
closed_at DATETIME 종료 시각
eligible_count INT 투표 행사 가능 수 (전환 시 확정)
last_sync_at DATETIME, NULL 위임장 마지막 성공 동기화 시각

7-2. Agenda (안건)

컬럼 타입 설명
id INT, PK 안건 고유 ID
assembly_id INT, FK 총회 ID
title VARCHAR 안건명
sort_order INT 표시 순서
status ENUM 'draft' / 'open' / 'closed'

7-3. AgendaOption (안건 보기)

컬럼 타입 설명
id INT, PK 보기 고유 ID
agenda_id INT, FK 안건 ID
label VARCHAR 보기 텍스트 (예: 찬성, 반대, 기권)
sort_order INT 표시 순서

7-4. Delegation (위임)

컬럼 타입 설명
id INT, PK 위임 고유 ID
assembly_id INT, FK 총회 ID
delegator_member_id VARCHAR 위임인 회원번호 (회원 DB 참조)
delegator_name VARCHAR 위임인 이름
delegator_phone VARCHAR 위임인 전화번호
delegatee_member_id VARCHAR, NULL 수임인 회원번호 (NULL이면 의장 귀속)
delegatee_name VARCHAR, NULL 수임인 이름
delegatee_phone VARCHAR, NULL 수임인 전화번호
status ENUM 'active' / 'cancelled' / 'invalid_match' / 'invalid_same' / 'invalid_duplicate' / 'invalid_grade'
cancelled_reason VARCHAR, NULL 취소 사유 (예: 'delegator_attended')
sheet_row INT 스프레드시트 원본 행 번호 (추적용)
synced_at DATETIME 마지막 동기화 시각

인덱스 권장: (assembly_id, delegatee_member_id) — 수임인 투표 화면에서 위임 목록 조회 성능 확보. (assembly_id, delegator_member_id) — 위임 취소 판정 및 중복 검증용.

7-5. Vote (투표)

컬럼 타입 설명
id INT, PK 투표 고유 ID
agenda_id INT, FK 안건 ID
option_id INT, FK 선택한 보기 ID
voter_member_id VARCHAR 투표권 소유자 회원번호 (위임인 또는 본인)
voted_by VARCHAR 실제 투표 행위자 식별자
· 본인 투표: 본인 회원번호
· 수임인 투표: 수임인 회원번호
· 의장(관리자) 일괄 투표: 관리자 계정 ID
voted_at DATETIME 투표 시각

투표 저장 방식: (agenda_id, voter_member_id)에 UNIQUE 제약 조건을 설정한다. 최초 투표 시 INSERT를 수행한다. UNIQUE 제약으로 INSERT가 실패하면, 안건이 open 상태인 경우에 한해 해당 레코드의 option_id와 voted_at을 UPDATE한다. 안건이 closed 상태이면 INSERT·UPDATE 모두 차단하고 "투표 마감된 안건입니다." 메시지를 표시한다. 두 기기에서 동시 제출 시, 먼저 도착한 요청이 INSERT 성공하고 나중 요청은 UPDATE로 처리되어 마지막 선택이 최종 반영된다.

7-6. Admin (관리자)

컬럼 타입 설명
id INT, PK 관리자 고유 ID
login_id VARCHAR 관리자 로그인 ID (= 의장)
password_hash VARCHAR 비밀번호 해시

8. 비기능 요구사항

항목 요구사항
반응형 투표자 화면은 모바일 우선 반응형 / 관리자 화면은 PC 우선 반응형
브라우저 Chrome, Safari, Samsung Internet 최신 버전 지원
동시접속 100~130명 동시 투표 처리 가능
폴링 관리자 현황 화면 10초 간격 자동 갱신
보안 HTTPS 필수. 관리자 세션 토큰 기반 인증. 서비스 계정 키 서버 환경변수 보관.
세션 투표자: 총회 종료 시까지 세션 유지. 관리자: 별도 로그인.

8-A. 투표 익명성 정책

Vote 테이블에는 voter_member_id와 선택 보기가 매핑되어 저장된다(투표 여부 확인 및 중복 투표 방지 목적). 단, 관리자 화면을 포함한 모든 UI에서 "누가 어떤 보기를 선택했는지"는 표시하지 않는다. 관리자가 확인할 수 있는 정보는 "투표함 / 미투표" 여부까지만이다.

DB 직접 조회 권한을 가진 인원에 한해 매핑 확인이 가능하므로, DB 접근 권한 관리는 운영 정책으로 통제한다.

8-B. 데이터 보관 정책

종료된 총회 데이터는 DB에 영구 보관하며, 관리자 화면에서 조회만 가능하다. 수정·삭제는 불가하다. 총회 목록에서 종료된 총회는 "종료" 뱃지와 함께 표시되며, 상세 화면 진입 시 모든 입력 기능이 비활성화된 읽기 전용 모드로 표시된다.

심각도 리스크 항목 영향 대응 방안
