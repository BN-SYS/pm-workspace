## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| 정회원 | 쇼핑몰 회원 등급이 정회원 이상인 회원. 총회 투표권의 대상자. |
| 티켓구매자 | 총회 현장 참석을 위해 티켓을 구매한 정회원. 티켓 구매는 현장 출석의 전제 조건이며, 투표권 발생의 전제 조건은 아니다. |
| 투표권 | 정회원 이상 회원 전원이 보유하는 안건별 1회의 의결 참여 권리. 행사 경로는 (1) 현장 출석(티켓 구매 + 검표) 또는 (2) 위임장 제출이다. |
| 투표 행사권 | 투표권을 실제로 행사할 수 있는 상태. 출석(검표 완료) 시 본인에게, 위임 시 수임인에게 발생한다. |
| 위임인 | 위임장을 제출하여 자신의 투표 행사권을 타인에게 이전하는 정회원. 티켓 미구매 회원도 가능. |
| 수임인 | 위임장을 통해 타인의 투표 행사권을 위임받아 대리 투표하는 정회원. |
| 의장 | 총회를 주재하는 관리자. 시스템에서는 관리자 계정 = 의장이다. 수임인이 지정되지 않은 위임건은 의장에게 자동 귀속된다. |
| 총회 상태 | 준비중 -> 투표 진행중 -> 종료. 각 전환은 관리자가 수동으로 진행한다. |
| 검표 | 현장에서 티켓 구매자의 출석을 확인하는 행위. 쇼핑몰 DB에 검표 시각(check_time)이 기록된다. |

### 2-1. 투표 행사권 발생 조건 상세

| 상황 | 투표권 | 투표 행사권 | 투표율 모수 포함 | 누가 투표하나 |
|---|---|---|---|---|
| 정회원 + 티켓 구매 + 검표 완료 + 위임 X | 있음 | 본인에게 발생 | **포함** | 본인 |
| 정회원 + 티켓 미구매 + 위임 O (수임인 지정) | 있음 | 수임인에게 이전 | **포함** | 수임인 |
| 정회원 + 티켓 미구매 + 위임 O (수임인 공란) | 있음 | 의장(관리자)에게 귀속 | **포함** | 의장 (관리자 일괄 투표) |
| 정회원 + 티켓 구매 + 미검표 + 위임 O (수임인 지정) | 있음 | 수임인에게 이전 | **포함** | 수임인 |
| 정회원 + 티켓 구매 + 미검표 + 위임 O (수임인 공란) | 있음 | 의장(관리자)에게 귀속 | **포함** | 의장 (관리자 일괄 투표) |
| 정회원 + 위임 O + "투표 진행중" 전환 시점에 검표 완료 상태 | 있음 | 위임 자동 취소 -> 본인 복귀 | **포함** | 본인 |
| 정회원 + 위임 O + "투표 진행중" 전환 이후 검표 | 있음 | 위임 유지 | **포함** (수임인 기준) | 수임인 |
| 정회원 + 티켓 미구매 + 위임 X + 불참 | 있음 | 행사 불가 | **제외** | 없음 |
| 정회원 + 티켓 구매 + 미검표 + 위임 X | 있음 | 행사 불가 | **제외** | 없음 |

> **모수 기준 원칙:** 투표 행사 가능 수(모수) = 출석 본인(검표 완료 + 위임 X) + 유효 위임(수임인 지정 + 의장 귀속). 이 외의 모든 경우는 모수에서 제외한다.
>
> **운영 권장:** "투표 진행중" 전환 시점 이전에 충분한 입장·검표 시간을 확보해 현장 참석자가 최대한 본인 투표를 할 수 있도록 한다.

## 4. 시스템 흐름 · 데이터 연동

### 4-1. 전체 흐름

| 단계 | 관리자 행위 | 시스템 동작 |
|---|---|---|
| 1. 총회 생성 | 총회명, 일시 입력 | Assembly 레코드 생성, 상태 = 준비중 |
| 2. 안건 등록 | 안건명 + 보기(선택지) 입력 | Agenda + AgendaOption 레코드 생성 |
| 3. 위임장 동기화 | 동기화 버튼 클릭 | 구글 스프레드시트 API -> Delegation 테이블 UPSERT |
| 4. 출석 현황 확인 | 출석 관리 화면 조회 | 쇼핑몰 DB 직접 참조 (티켓 + 검표 시각) |
| 5. 투표 시작 | "투표 진행중" 전환 (더블 컨펌) | (1) 강제 동기화 1회 -> (2) 검표 시각 기준 위임 취소 판정 -> (3) 모수 확정 -> (4) 스프레드시트 연동 차단 |
| 6. 안건 열기 | 안건별 "열기" 클릭 | 해당 안건 투표 가능 상태로 전환 |
| 7. 투표 진행 | 현황 모니터링 | 투표자 화면에서 선택지 제출 -> Vote 레코드 생성 (열림 중 수정 가능) |
| 8. 안건 닫기 | 안건별 "닫기" 클릭 (더블 컨펌) | 해당 안건 투표 마감, 결과 집계 |
| 9. 총회 종료 | "종료" 클릭 (더블 컨펌) | 전체 투표 마감, 최종 결과 확정. 되돌릴 수 없음. |

> **투표자 인증 -> 투표 흐름:** 이름+전화번호+주문번호 뒤 5자리 입력 -> 티켓 DB 매칭 -> 성공 시 회원번호를 세션에 저장 -> Delegation 테이블에서 delegatee_member_id로 위임 목록 조회 -> 위임건이 있으면 수임인 드롭다운 구성 -> 안건별 투표 진행.

### 4-2. 데이터 연동 구조

#### 4-2-A. 출석 관리 (쇼핑몰 DB 직접 참조)

| 항목 | 내용 |
|---|---|
| 연동 방식 | 같은 서버 내 쇼핑몰 DB 직접 쿼리 (별도 API 불필요) |
| 조회 대상 | 티켓 구매 테이블 + 검표 시각(check_time) 컬럼 |
| 데이터 갱신 | 관리자 화면에서 실시간 조회 (페이지 로드 또는 새로고침 시 DB 재조회) |
| 표시 항목 | 구매자명, 전화번호, 주문번호, 검표 여부, 검표 시각 |

#### 4-2-B. 위임장 관리 (구글 스프레드시트 연동)

| 항목 | 내용 |
|---|---|
| 연동 방식 | Google Sheets API v4 + 서비스 계정 인증 (서버 사이드) |
| 스프레드시트 컬럼 | A: 위임자명 / B: 위임자휴대폰 / C: 위임자회원번호 / D: 수임자명 / E: 수임자휴대폰 / F: 수임자회원번호 |
| 매칭 기준 | 위임자: 회원번호(C열)로 회원 DB 매칭 / 수임자: 회원번호(F열)로 회원 DB 매칭 |
| 동기화 방식 | 준비중: 수동 반복 / 전환 시: 강제 1회 -> 이후 차단 |
| 수임인 공란 처리 | D·E·F열 모두 공란이면 의장(관리자)에게 자동 귀속 |
| 보안 | 서비스 계정 JSON 키는 서버 환경변수에 보관. 프론트엔드 노출 금지. |

## 5. 관리자 기능 상세

### 5-1. 총회 관리
- 총회 생성: 총회명, 일시 입력. 생성 시 상태 = "준비중".
- 총회 목록: 생성된 총회 목록 표시. 상태별 필터 가능.

### 5-2. 안건 관리
- 안건 등록: 안건명 입력 + 보기(선택지) 최소 2개 등록. **"준비중" 및 "투표 진행중"** 상태에서 가능. "종료" 후 불가.
- 안건 수정·삭제: **"준비중"** 상태에서만 가능. 투표 진행중 이후 수정·삭제 불가.
- 안건 열기·닫기: **"투표 진행중"** 상태에서 안건별 개별 토글. 열린 안건만 투표 가능.

| 행위 | 준비중 | 투표 진행중 | 종료 |
|---|---|---|---|
| 안건 추가 | 가능 | **가능** | 불가 |
| 안건 수정 | 가능 | 불가 | 불가 |
| 안건 삭제 | 가능 | 불가 | 불가 |
| 안건 열기·닫기 | 불가 | 가능 | 불가 |

> **긴급 안건 참고:** 투표 진행중에 추가된 안건은 기존 확정된 모수를 동일하게 적용한다. 추가 후 관리자가 "열기"를 해야 투표가 시작되므로 기존 진행 중인 안건에 영향을 주지 않는다.

> **보기(선택지) 수정 규칙:** 투표 진행중에 추가된 안건은 **"열기" 전까지 보기 수정이 가능**하다. 한 번 "열림" 상태가 되면 보기 수정·삭제 불가.

### 5-3. 총회 상태 관리

| 전환 | 조건 | 시스템 동작 |
|---|---|---|
| 준비중 -> 투표 진행중 | 더블 컨펌 | (1) 강제 동기화 -> (2) 위임 취소 판정 -> (3) 모수 확정 -> (4) 스프레드시트 차단 |
| 투표 진행중 -> 종료 | 더블 컨펌 | 모든 안건 강제 닫힘. 최종 결과 확정. 되돌릴 수 없음. |

> **롤백 불가:** MVP에서 롤백 미지원. "종료"도 되돌릴 수 없다.

> **안건 가드:** 안건 0건이면 "투표 진행중" 전환 차단. "안건을 1개 이상 등록하세요." 알림.

> **안건 닫기 정책:** 더블 컨펌 수행.
> 1차: "해당 안건의 투표를 마감합니다. 닫으시겠습니까?"
> 2차: "닫힌 안건은 재오픈할 수 있지만, 재오픈 시 미투표자의 추가 투표로 결과가 변동될 수 있습니다. 정말 닫으시겠습니까?"

> **안건 재오픈 정책:** 기존 투표 유지. 미투표자만 추가 투표. 재오픈 횟수 제한 없음. 열기·닫기 이력 로그 기록.

### 5-4. 투표 현황

| 표시 항목 | 설명 |
|---|---|
| 전체 정회원 수 | 회원 DB 기준. 쿼리 불가 시 "미제공" 표시. |
| 티켓 구매자 수 | 현장 참석 티켓 구매 회원 수 |
| 출석 본인 수 | 검표 완료 + 위임 X 인원 |
| 유효 위임 수 | 수임인 지정 + 의장 귀속 합산 |
| 불참·무위임 수 | 참고용 (모수 제외) |
| 투표 행사 가능 수 (모수) | 출석 본인 + 유효 위임 |
| 투표 완료 수 | 본인 + 수임인 + 의장 일괄 합산 |
| 미투표 수 | 행사 가능 수 - 완료 수 |
| 투표율 | 완료 수 / 행사 가능 수 x 100 |

> **투표율 산식:** 모수는 "투표 진행중" 전환 시점 기준 고정.
> **정족수 판정:** MVP 미포함. 결과만 표시, 정족수 충족은 의장 별도 판단.
> **익명성 원칙:** 안건별 투표/미투표 수만 표시. 개별 선택 보기는 관리자 포함 모든 UI에서 미표시. 10초 폴링.

### 5-5. 결과 조회

| 항목 | 설명 |
|---|---|
| 표시 시점 | 안건 closed 전환 즉시 관리자 화면에서 확인 가능 |
| 표시 항목 | 보기별 득표수, 보기별 득표율, 총 투표 수, 모수 대비 투표율 |
| 정족수 판정 | MVP 미포함. 숫자만 표시, 가결/부결은 의장 별도 판단. |
| 투표자 화면 | MVP에서 결과 미표시. 투표 완료 여부 + 선택 보기만 확인 가능. |

### 5-6. 출석 관리
- 출석 현황 조회: 쇼핑몰 DB에서 티켓 구매자 + 검표 시각을 직접 조회하여 표시
- 표시 정보: 구매자명, 전화번호, 주문번호, 검표 여부(Y/N), 검표 시각
- 위임 여부 표시: 각 출석자 옆에 위임 상태 표시

### 5-7. 위임장 관리

#### 5-7-A. 위임 검증 규칙

| # | 검증 항목 | 조건 | 실패 시 처리 |
|---|---|---|---|
| 1 | 위임자 자격 검증 | 회원번호(C열) 회원 DB 매칭 + 총회 당일 기준 정회원 이상 | 매칭 실패 또는 위임자 자격 미달 표시 |
| 2 | 수임인 공란 -> 의장 귀속 | D·E·F열 모두 공란 | 의장(관리자)에게 자동 귀속 |
| 3 | 위임인 출석 여부 | 위임에는 출석(검표) 불필요 | - |
| 4 | 위임인 = 수임인 동일인 | C열 = F열 | 무효(동일인) |
| 5 | 1인당 수임 건수 제한 | 제한 없음 | 카운터 표시 |
| 6 | 동일 위임자 중복 등록 | C열 2행 이상 | 마지막 행만 유효, 이전 행 무효(중복) |
| 7 | 수임인 자격 검증 | 회원번호(F열) 매칭 + 정회원 이상 | 수임인 자격 미달 표시, 보류 |

#### 5-7-B. 위임 취소 판정 (투표 시작 시)

| 조건 | 처리 |
|---|---|
| 전환 시점에 위임인의 check_time 존재 + check_time < 전환 시각 | 위임 자동 취소 (cancelled_reason: delegator_attended) -> 위임인 본인에게 투표 행사권 귀속 |
| 전환 이후에 위임인이 검표 | MVP에서는 위임 유지 -> 수임인 계속 투표. 전환 이후 검표이면 위임 취소하지 않음. |

> **판정 기준:** 쇼핑몰 DB의 원천 check_time 기준. 전환 시 강제 동기화 직후 일괄 판정.

#### 5-7-C. 위임 데이터 반영 컷오프

| 총회 상태 | 스프레드시트 동기화 | 위임 데이터 변경 |
|---|---|---|
| 준비중 | 가능 (수동 반복) | 반영됨 |
| 투표 진행중 (전환 시) | 강제 1회 | 최종 반영 |
| 투표 진행중 (전환 후) | 차단 | 변경 불가 |
| 종료 | 차단 | 변경 불가 |

#### 5-7-D. 스프레드시트 동기화 실패 처리

> **상태 유지 원칙:** 동기화 실패 시 "준비중" 유지. CSV 업로드 등 준비중 기능 계속 사용 가능.
>
> **기본 동작:** 전환 시 강제 동기화 실패 -> 자동 최대 3회 재시도. 3회 실패 시 전환 차단 + "위임장 동기화에 3회 실패했습니다. CSV 업로드를 이용해주세요." + CSV 업로드 버튼 표시.
>
> **비상 전환:** CSV도 불가 시 추가 선택지 제공. 마지막 성공 데이터 있으면 해당 데이터로 전환 가능. 없으면 "위임 데이터 없이 투표를 시작하시겠습니까?" 안내.

#### 5-7-E. CSV 업로드 (API 장애 시 백업)

| 항목 | 설명 |
|---|---|
| 용도 | 구글 스프레드시트 API 연동 실패 시 백업 경로 |
| 절차 | 스프레드시트에서 CSV 다운로드 -> 위임장 관리 화면에서 업로드 |
| 파일 형식 | UTF-8 CSV. 컬럼 순서 A~F 동일 |
| 검증 | 7개 검증 규칙 동일 적용 |
| 처리 | 기존 데이터 전체 교체 (UPSERT). 업로드 시각 = 마지막 동기화 시각 |
| 사용 가능 상태 | "준비중"에서만. "투표 진행중" 이후 차단. |

> **CSV 업로드 오류 처리:** UTF-8 아닌 경우, 컬럼 6개 아닌 경우, 빈 파일인 경우 각각 에러 메시지 표시. 검증 실패 행은 정상 행과 분리 표시.

### 5-8. 의장 귀속 위임 일괄 투표

> **실행 조건:** 총회 "투표 진행중" + 안건 "open" 상태에서만 가능. 닫힘/준비중이면 비활성화.
> **수정:** 안건 open 동안 수정 가능. closed이면 불가. 재오픈 시 미투표 건 추가 + 기투표 건 수정 가능.

| 항목 | 설명 |
|---|---|
| 대상 | 수임인 미지정 -> 의장 귀속 위임건 |
| 투표 방식 | 안건별 보기 선택 -> "일괄 투표" 버튼 |
| 일괄 처리 | 전체 귀속 건에 동일 보기 적용 (건별 선택은 v2) |
| DB 기록 | voter_member_id별 개별 Vote 레코드. voted_by = 관리자 ID. |
| 표시 | 위임인 이름 옆에 '투표함 (위임 - 의장 귀속)' |

## 6. 투표자 기능 상세

### 6-1. 본인 인증
- 인증 방식: 이름 + 전화번호 + 주문번호 뒤 5자리 (별도 인증)
- 검증: 쇼핑몰 DB 티켓 구매 테이블에서 3개 값 일치 확인
- 실패 시: "일치하는 정보가 없습니다."
- 인증 후: 회원번호를 세션에 저장. Delegation 테이블에서 위임 목록 조회 -> 드롭다운 자동 구성.
- 세션 유지: 총회 종료 시까지 유지. 종료 시 모든 투표자 세션 만료 -> "총회가 종료되었습니다." 안내. 재접속 시 세션 유효하면 자동 복원.
- 재인증 시 복원: DB 조회하여 기투표 안건 "투표 완료" 상태 복원.

> **향후 개선:** 쇼핑몰 로그인 세션 활용으로 별도 입력 제거 검토 (v2).

### 6-2. 투표 화면
- 안건 목록: 모든 안건을 상태와 함께 표시.
  - 준비중(draft): "예정" 뱃지 + 보기 목록 표시(비활성화). 투표 불가. "안건 준비 중 - 보기(선택지)가 변경될 수 있습니다." 안내.
  - 열림(open): 라디오 버튼 활성화. 투표 가능.
  - 닫힘(closed): "마감" 뱃지 + 보기 목록 표시(비활성화) + 기존 선택 내역 표시. 투표 불가. 결과 미표시.
- 투표 제출: 선택 -> "투표" 버튼 -> 확인 팝업: "투표를 제출합니다. 안건이 열려 있는 동안 수정할 수 있습니다." -> 제출.
- 투표 수정: 안건 "열림" 동안 자유롭게 수정 가능. "수정" 버튼 표시. 닫히면 불가.
- 투표 완료: "투표 완료" 뱃지 + 선택 보기 + "수정" 버튼.
- 제출 실패: "투표 저장에 실패했습니다. 다시 시도해주세요." UNIQUE 제약으로 중복 방지.

| 안건 상태 | 관리자 화면 | 투표자 화면 |
|---|---|---|
| draft | 표시 + 수정·삭제 | 표시 + "예정" 뱃지 + 보기 표시(비활성화) + "보기 변경 가능" 안내 |
| open | 표시 + 닫기 | 표시 + 투표 + 수정 |
| closed | 표시 + 재오픈 + 결과 즉시 표시 | 표시 + "마감" 뱃지 + 보기 표시(비활성화) + 기존 선택 내역 표시 + 결과 미표시 |

> **브라우저 새로고침 안내:** 제출 전 새로고침/이탈 시 선택 미보존. 제출 완료된 투표는 DB 조회로 복원. 화면 상단에 "투표는 제출 후에만 저장됩니다. 새로고침 시 미제출 선택은 초기화됩니다." 안내.

#### 6-2-A. 수임인 투표 화면 구조

| 항목 | 설명 |
|---|---|
| 레이아웃 | 전체 진행 요약 + 드롭다운 화면 상단 고정(sticky) |
| 전체 진행 요약 | 예: '전체 진행: 8/15 완료 (3명 중 1명 미완료)' |
| 드롭다운 구성 | 첫 항목: 본인 명의 + 진행률. 이후: 위임인별 + 진행률. 미완료 상단 정렬. |
| 동작 방식 | 선택 -> 해당 명의 안건·투표 상태 표시. 명의별 독립 투표. |
| 완료 표시 | 미완료: '2/5 미완료' / 완료: '5/5 완료' |
| 이탈 경고 | 미완료 명의 있으면 "위임 투표 - 김위임 외 N건 미완료" 알림 |
| 제한 | 의장 귀속 건은 수임인 드롭다운 미표시 (관리자 전용) |

> 종료된 총회 데이터는 DB에 영구 보관. 관리자 화면에서 조회만 가능. 수정·삭제 불가. "종료" 뱃지 + 읽기 전용 모드.
