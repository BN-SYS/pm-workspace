## 1. 프로젝트 개요

### 1-1. 목적

총회 현장에서 안건별 단일선택 전자투표를 진행하고, 투표 종료 즉시 결과를 집계하여 관리자에게 제공하는 시스템을 구축한다. 시스템은 **쇼핑몰 서버 내부에 구축**을 전제로 하여 회원 DB 및 검표 데이터를 직접 조회하도록 설계한다.

### 1-2. MVP 범위

| 포함 (MVP) | 제외 (v2 이후) |
|---|---|
| 관리자: 총회 등록, 안건 CRUD, 투표 열기/닫기, 총회 상태 관리, 의장 일괄 투표 | 복수선택 투표 |
| 투표자: 본인 인증 → 안건 목록 → 투표 | 자체 위임 등록 화면 |
| 위임: 구글 스프레드시트 기반 위임장 연동, 의장(관리자) 자동 귀속 | 투표자용 별도 결과 화면 |
| 출석: 쇼핑몰 검표 DB 직접 조회 (동일 서버) | 카카오 알림톡, 다중 관리자 권한 |
| 결과: 안건별 득표율, 참여자 명단(익명 투표 원칙 준수) | 의장 건별 개별 선택, 감사 로그, 아카이브 |

### 1-3. 주요 수치

| 항목 | 값 |
|---|---|
| 예상 참여자 수 | 100~130명 |
| 투표 방식 | 안건별 단일선택 |
| 서버 환경 | 쇼핑몰 동일 서버 (PHP 백엔드, DB 직접 접근) |
| 목표 오픈 | 2026-04-25 |

## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| 총회 (Assembly) | 투표가 진행되는 단위 행사. 상태: 준비중 → 투표 진행중 → 종료 |
| 안건 (Agenda) | 총회 내 개별 투표 항목. 상태: 닫힘(기본) ↔ 열림 |
| 보기 (Option) | 안건 내 선택지. 개수 제한 없음. |
| 회원번호 | 쇼핑몰 회원 고유 식별자. 위임 매칭 및 투표권 귀속의 기준 키. |
| 정회원 | 쇼핑몰 회원 등급이 정회원 이상인 회원. 총회 투표권의 대상자. |
| 티켓구매자 | 총회 현장 참석을 위해 티켓을 구매한 정회원. 티켓 구매는 현장 출석의 전제 조건이며, 투표권 발생의 전제 조건은 아니다. |
| 투표권 | 정회원 이상 회원 전원이 보유하는 안건별 1회의 의결 참여 권리. 행사 경로는 ① 현장 출석(티켓 구매 + 검표) 또는 ② 위임장 제출이다. |
| 투표 행사권 | 투표권을 실제로 행사할 수 있는 자격. ① 본인 행사: 티켓 구매 + 현장 검표 완료 시 ② 위임 행사: 위임장을 통해 수임인에게 이전 시 (위임인 출석 불요, 티켓 미구매도 가능) |
| 위임인 | 투표 행사권을 타인에게 위임하는 정회원. 티켓 미구매 회원도 위임 가능. |
| 수임인 | 위임을 받아 대신 투표를 행사하는 회원. 수임인 본인의 출석(검표)은 불요. |
| 의장 | 관리자 = 의장. 별도 회원번호 없음. 수임인 공란 시 의장에게 자동 귀속, 관리자 페이지에서 일괄 투표 행사. |
| 검표 | 현장 QR 스캔을 통한 출석 확인. 쇼핑몰 시스템에서 처리, 결과는 동일 DB에 기록. |

### 2-1. 투표 행사권 발생 조건 상세

| 상황 | 투표권 | 투표 행사권 | 투표율 모수 포함 | 누가 투표하나 |
|---|---|---|---|---|
| 정회원 + 티켓 구매 + 검표 완료 + 위임 X | 있음 | 본인에게 발생 | 포함 | 본인 |
| 정회원 + 티켓 미구매 + 위임 O (수임인 지정) | 있음 | 수임인에게 이전 | 포함 | 수임인 |
| 정회원 + 티켓 미구매 + 위임 O (수임인 공란) | 있음 | 의장(관리자)에게 귀속 | 포함 | 의장 (관리자 일괄 투표) |
| 정회원 + 티켓 구매 + 위임 O (수임인 지정) + 투표 시작 전 검표 | 있음 | 위임 취소 → 본인 복귀 | 포함 | 본인 |
| 정회원 + 티켓 구매 + 위임 O + 투표 시작 후 검표 | 있음 | 위임 유지 | 포함 (수임인 기준) | 수임인 |
| 정회원 + 티켓 미구매 + 위임 X + 불참 | 있음 | 행사 불가 | **제외** | 없음 |
| 정회원 + 티켓 구매 + 미검표 + 위임 X | 있음 | 행사 불가 | **제외** | 없음 |

> **모수 기준 원칙:** 투표 행사 가능 수(모수) = 출석 본인 + 유효 위임(수임인 지정 + 의장 귀속). 이 외의 모든 경우(불참·무위임, 티켓 구매했지만 미출석·무위임)는 모수에서 제외한다.

## 4. 시스템 흐름 및 데이터 연동

### 4-1. 전체 시스템 흐름도

\`\`\`
┌──────────────────────────────────────────────┐
│              쇼핑몰 서버 (동일 서버)            │
│                                                │
│  [회원 DB] [주문/티켓 DB]                      │
│      │           │                             │
│      │ 직접 조회  │ 직접 조회                   │
│      ▼           ▼                             │
│  ┌────────────────────────────────┐           │
│  │     전자투표 시스템 (신규 구축)   │           │
│  │  [위임 DB] [투표 DB]            │           │
│  │  [관리자 페이지] [투표자 페이지]  │           │
│  └────────────────────────────────┘           │
│                                                │
│  [현장 QR 리더기] → [쇼핑몰 검표] → 주문/티켓 DB │
└──────────────────────────────────────────────┘
         ↕ Google Sheets API (준비중 상태에서만)
  [구글 스프레드시트 - 시트명: "위임장"]
\`\`\`

### 4-2. 데이터 연동

#### A. 쇼핑몰 DB 직접 조회

동일 서버에 구축하므로 외부 API 없이 DB 직접 SELECT로 처리한다.

| 항목 | 내용 |
|---|---|
| 조회 방식 | 동일 서버 DB 직접 SELECT (실시간) |
| 회원 DB 조회 항목 | 회원번호, 이름, 전화번호, 회원등급 |
| 주문/티켓 DB 조회 항목 | 주문번호, 회원번호(FK), 검표 상태(Y/N), 검표 시각(timestamp) |
| 현장 구매 대응 | 동일 DB에 즉시 반영되므로 별도 동기화 불필요 |

#### B. 위임 데이터 (구글 스프레드시트)

위임장은 외부 시스템에서 수합하여 구글 스프레드시트에 정리한다. 관리자가 동기화 버튼으로 Google Sheets API를 통해 투표 DB에 반영한다. 수임인 정보가 모두 공란인 경우, 의장(관리자)에게 자동 귀속된다.

| 항목 | 내용 |
|---|---|
| 시트명 | "위임장" |
| 시트 컬럼 | A: 위임자명 / B: 위임자휴대폰 / C: 위임자회원번호 / D: 수임자명 / E: 수임자휴대폰 / F: 수임자회원번호 |
| 매칭 기준 | 위임자: 회원번호(C열)로 회원 DB 매칭 (티켓 미구매 회원도 위임 가능) / 수임자: 회원번호(F열)로 회원 DB 매칭 |
| 수임자 공란 처리 | D·E·F 모두 공란 → 의장(관리자)에게 자동 귀속 |
| 동기화 방식 | 관리자 수동 동기화 버튼 (Google Sheets API v4) |

#### 스프레드시트 API 통신 관리

| 총회 상태 | API 통신 | 비고 |
|---|---|---|
| 준비중 | ✅ 허용 | 동기화 가능 |
| 투표 진행중 | ❌ **차단** | 위임 데이터 동결 |
| 종료 | ❌ **차단** | 변경 불가 |

## 5. 관리자 기능 상세

### 5-1. 로그인

| 항목 | 내용 |
|---|---|
| 인증 방식 | ID / PW 별도 관리자 계정 (쇼핑몰 회원 계정과 분리) |
| 역할 | 관리자 = 의장 (별도 회원번호 없음) |
| 세션 유지 | 브라우저 세션 기반 |

### 5-2. 총회 등록·관리

| 항목 | 내용 |
|---|---|
| 총회명 | 필수 입력 |
| 일시 | 필수 입력 |
| 설명 | 선택 입력 |
| 상태 | 자동: 준비중 |

> 의장은 관리자 본인이므로 별도 입력 필드가 없다. 수임인 공란 시 의장(관리자)에게 자동 귀속.

### 5-3. 총회 상태 관리

| 상태 | 전환 조건 | 효과 |
|---|---|---|
| 준비중 | 등록 시 기본값 | 안건 CRUD 가능. 위임 동기화 가능. 투표 불가. |
| 투표 진행중 | 관리자 수동 전환 (더블 컨펌) | ① 스프레드시트 강제 동기화 1회 ② API 통신 차단 ③ 위임 취소 판정 ④ 투표 시작 시각 기록 ⑤ 안건 토글·투표 가능 ⑥ 의장 일괄 투표 가능 |
| 종료 | 관리자 수동 전환 | 모든 안건 강제 닫힘. 결과 확정. |

#### "투표 진행중" 전환 더블 컨펌

| 단계 | 팝업 내용 |
|---|---|
| 1차 | "총회 상태를 '투표 진행중'으로 변경합니다. 계속하시겠습니까?" |
| 2차 | "투표 진행으로 변경 시, 이후 위임인 본인이 출석하여도 수임인에게 투표권이 유지됩니다. 투표를 시작하시겠습니까?" |

> 안건 재오픈 시 "결과가 변동될 수 있습니다" 경고 팝업 표시. 종료는 되돌릴 수 없다. "투표 진행중 → 준비중" 롤백도 MVP에서는 지원하지 않는다.

### 5-4. 안건 관리

| 항목 | 내용 |
|---|---|
| 안건명 | 필수 입력 |
| 안건 설명 | 선택 입력 |
| 보기 | 무제한 추가 가능 (단일선택) |
| 안건 번호 | 자동 부여 |
| 상태 | 기본 닫힘, 자유 토글 |

#### 투표율 산식

| 구분 | 정의 |
|---|---|
| 투표 행사 가능 수 (모수) | 출석 본인 + 유효 위임(수임인 지정 + 의장 귀속). 불참·무위임 제외. |
| 투표 완료 수 | 실제 투표 제출 수 (본인 + 위임 + 의장 일괄 합산) |
| 투표율 | (투표 완료 수 / 투표 행사 가능 수) × 100 |

> 모수는 "투표 진행중" 전환 시점에 확정, 이후 변동 없음.

#### 관리자 결과 화면 통계

| 표시 항목 | 설명 |
|---|---|
| 전체 정회원 수 | 참고 수치 (총회 투표권 대상자 전체) |
| 전체 티켓 구매자 | 참고 수치 (현장 참석 신청자) |
| 출석 본인 | 검표 완료 + 위임 없음 (또는 위임 취소 후 본인 복귀) |
| 유효 위임 (수임인 지정) | 수임인 지정 건수 |
| 유효 위임 (의장 귀속) | 수임인 공란 → 의장 귀속 건수 |
| 불참·무위임 | 모수에서 제외 |
| **투표 행사 가능 수** | 출석 본인 + 유효 위임 전체 |
| 투표 완료 | 실제 제출 수 |
| 미투표 | 행사 가능 − 완료 |
| **투표율** | (완료 / 행사 가능 수) × 100 |

### 5-5. 출석 관리 (쇼핑몰 검표 DB 직접 조회)

동일 서버 DB를 직접 조회하므로 별도 동기화 불필요.

#### 조회 대상 데이터

| 필드 | 설명 |
|---|---|
| 주문번호 | 티켓 식별 키 |
| 회원번호 | 회원 DB 참조 키 |
| 구매자 이름 | 회원 DB 조회 |
| 구매자 전화번호 | 회원 DB 조회 |
| 검표 상태 | Y / N |
| 검표 시각 | 위임 취소 판정 기준 |

#### 관리자 출석 화면

| 표시 항목 | 설명 |
|---|---|
| 전체 티켓 수 | 총 발행 수 |
| 출석 확인 수 | 검표 Y |
| 미출석 수 | 검표 N |
| 참석자 목록 | 이름, 전화번호, 회원번호, 주문번호, 검표 시각 |
| 수동 출석 처리 | 예외 상황 대응용 |

### 5-6. 위임 관리

#### 위임 데이터 동기화

| 항목 | 내용 |
|---|---|
| 데이터 소스 | 구글 스프레드시트 (시트명: "위임장") |
| 시트 컬럼 | A: 위임자명 / B: 위임자휴대폰 / C: 위임자회원번호 / D: 수임자명 / E: 수임자휴대폰 / F: 수임자회원번호 |
| 동기화 방식 | 관리자 수동 동기화 (준비중 상태에서만) |
| 매칭 기준 | 회원번호로 회원 DB 매칭 (위임자: 티켓 미구매 회원도 위임 가능) |
| 수임자 공란 | D·E·F 모두 공란 → 의장 귀속 |

#### 수임인 유효성 검증

| # | 검증 항목 | 통과 조건 | 실패 시 처리 |
|---|---|---|---|
| 1 | 위임자 회원 DB 존재 | 회원번호(C열) 매칭 성공 + 정회원 이상 | 매칭 실패 |
| 2 | 수임자 회원 DB 존재 | 회원번호(F열) 매칭 성공 | 매칭 실패 |
| 3 | 수임자 공란 | D·E·F 모두 공란 | 의장 자동 귀속 |
| 4 | 수임자 검표 여부 | — | 검표 불요 |
| 5 | 위임자 = 수임자 동일인 | 회원번호 다른 사람 | 무효 |
| 6 | 수임 한도 초과 | ≤ 10건 | 한도 초과 |

> 상태 뱃지: 유효 / 매칭 실패 / 의장 귀속 / 무효 / 한도 초과

#### 위임 취소 판정 로직

"투표 진행중" 전환 시(더블 컨펌 후) 자동 실행:

1. 구글 스프레드시트 강제 동기화 1회
2. 스프레드시트 API 통신 차단
3. 위임 목록 순회, 위임인 검표 상태 확인 (DB 직접 조회)
4. 판정:

| 조건 | 처리 |
|---|---|
| 위임인 검표 시각이 전환 시각 이전 | 위임 자동 취소 → 본인 복귀 |
| 위임인 검표 시각이 전환 시각 이후 또는 미검표 | 위임 유지 |

> ⚠️ "투표 진행중" 전환 이후 출석한 위임인은 위임 유지. 수임인이 투표, 위임인 본인 투표 불가.

> "투표 진행중" 전환 시 실행되는 강제 동기화가 위임 데이터의 최종 반영 시점이다. 전환 이후에는 스프레드시트가 변경되더라도 시스템에 반영되지 않으며, API 통신 자체가 차단된다. 이미 투표가 행사된 위임 관계는 어떤 경우에도 소급 변경하지 않는다. MVP에서는 총회 상태 롤백을 지원하지 않는다.

#### 관리자 위임 화면

| 표시 항목 | 설명 |
|---|---|
| 유효 위임 (수임인 지정) | 수임인 지정 유효 건수 |
| 유효 위임 (의장 귀속) | 의장 귀속 건수 |
| 취소 위임 수 | 출석으로 취소된 건수 |
| 매칭 실패 수 | 수동 확인 필요 |
| 위임 목록 | 위임자 이름·회원번호, 수임자 이름·회원번호(또는 의장 귀속), 상태 뱃지 |
| 동기화 버튼 | 준비중에서만 활성화 |
| 최종 동기화 시각 | 마지막 동기화 일시 |

### 5-7. 의장 일괄 투표

수임인 미지정 → 의장 귀속된 위임 건을 관리자 페이지에서 안건별 일괄 투표.

| 항목 | 내용 |
|---|---|
| 대상 | 의장 귀속 위임 건 전체 |
| 투표 방식 | 안건별 보기 선택 → "일괄 투표" → 전체 건에 동일 보기 적용 |
| 투표 기록 | 위임인 각각의 투표로 개별 INSERT |
| 참여자 명단 | 위임인 이름으로 "투표함 / 위임 (의장 귀속)" 표시 |
| 수정 | 안건 열림 상태에서 재선택·재적용 가능 |
| 건별 개별 선택 | MVP 제외 (v2) |
