## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트 유형 | 자사 SaaS 서비스 소개 랜딩페이지 (MVP) |
| 서비스명 | Verihum |
| 핵심 기능 | 학생 글쓰기 과정 AI 분석 → 교사에게 리포트 제공 |
| 목표 | 학교·교사 대상 도입 문의 유도 |
| 도입 문의 URL | https://bn-system.com/it_sub/contact.php |
| 산출물 | `outputs/intro/index.html` (단일 HTML 파일) |

### 핵심 가치 제안

> "학생은 평소처럼 과제를 작성합니다. 제출 후 교사는 리포트로 작성 과정 전체를 확인할 수 있습니다."

- **학생 관점**: 기존 방식 그대로 글쓰기. 추가 도구 불필요.
- **교사 관점**: 제출 후 작성 과정 리포트 확인. AI 활용 여부 분석.

## 2. 타겟 및 시장 배경

| 항목 | 내용 |
|------|------|
| 주요 타겟 | 고등학교 교사, 학교 관리자 |
| 2차 타겟 | 수행평가·생기부 담당 교사, 교육청 |
| 시장 근거 | 고교생 **96.5%** AI 활용 (진학사 설문 351명, 2025.10.29) |
| 문제 인식 | AI 활용 글쓰기 증가 → 교사의 진위 판별 필요성 급증 |

> **통계 출처 고정**: 96.5% 수치는 진학사 2025년 조사 기준. 임의 변경 금지.
> 원문: https://www.kyobit.com/news/articleView.html?idxno=2078

## 3. 페이지 섹션 구성

- 학생이 실시간으로 보는 화면이 **아니다**.
- 레이블: "작성 과정 리포트 · 교사 검토"
