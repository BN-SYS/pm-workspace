## 기획 방향 요약

| 항목 | 확정 방향 |
|------|-----------|
| 결제 구조 | 기본결과 / 상세리포트 / 상세리포트+오답확인 3종, 오답확인 단독 결제 불가 |
| 업그레이드 | 하위 상품 구매자에게 상위 상품 차액 결제 유도 |

| 유출 방지 | 워터마크 삽입 (수험번호+이름), 드래그·캡처·우클릭 방지 |

- 오답 확인 **단독 결제 불가** — 반드시 상세리포트와 번들로만 구매 가능

- 결과지(기본결과·상세리포트)는 응시 시 입력한 이메일로 링크 자동발송
- 오답확인 화면은 이메일 발송 없이 결제 직후 웹 1회 열람만 가능

## 02. 사용자 흐름 — 결제 · 열람 · 업그레이드

### A. 최초 결제 흐름

> 테스트 완료 → 결제 페이지(3종 상품 선택) → 결제 완료 → 결과 화면 진입

### B. 결과지 접근 흐름 (기본결과 · 상세리포트)

> 결제 완료 → 응시 시 입력한 이메일로 결과지 링크 자동발송 → 링크 클릭 → **이메일 재입력 인증** → 결과 화면 열람


### C. 오답확인 열람 흐름 (상세리포트+오답확인 구매자 전용)

> 결제 완료 → **오답 확인 화면 즉시 노출 (웹 1회 열람)** → 열람 완료 후 "상세리포트 확인하기" 버튼 → 상세리포트 화면 이동

- 오답확인은 결제 직후 **오답 내역을 먼저** 보여줌
- 하단 "상세리포트 확인하기" 버튼을 통해 상세리포트로 이동
- 오답확인 화면 이탈 시 재접근 불가 (1회 열람 정책)
- 상세리포트는 이메일 링크를 통해 추후 재열람 가능

### D. 상위 상품 업그레이드 (차액 결제)

| 현재 구매 상품 | 결과 화면 하단 유도 영역 | 차액 |
|---------------|-------------------------|------|
| ① 기본결과 (14,900원) | 상세리포트 업그레이드 → **+9,000원** / 상세리포트+오답확인 업그레이드 → **+14,000원** | 9,000원 / 14,000원 |
| ② 상세리포트 (23,900원) | 상세리포트+오답확인 업그레이드 → **+5,000원** | 5,000원 |
| ③ 상세리포트+오답확인 (28,900원) | 유도 영역 없음 (최상위 상품) | — |

> ⚠️ **협의 필요** — 기본결과 구매자가 번들(28,900원)로 업그레이드 시 차액 14,000원 적용 여부는 클라이언트 최종 확인이 필요합니다.

## 03. 오답 문제 노출 범위

| 항목 | 내용 |
|------|------|
| 노출 항목 | 단계별 틀린 문항 수(00/00) 표시 후 해당 단계 문제 묶음 노출. 틀린 문제 이미지 및 내가 선택한 보기 이미지 제공. 문제 번호는 별도 표시 없이 단계 내 순차 번호로 대체 |
| 미노출 항목 | 정답 보기 이미지, 정답 표시 |
| 정렬 기준 | 단계별 → 문항 번호 순 (오름차순) |
| 단계 타이틀 | 관리자 설정값 표시 (예: 패턴인식, 논리사고, 종합인지) |
| 유출 방지 | 워터마크 삽입, 우클릭 방지, 캡처 방지, 드래그 방지 / 안내 문구 노출 |

### 워터마크 규격

- 구성: 응시번호 + 이름
- 방식: 대각선 반복 삽입 (투명도 15~20%)
- 처리: 서버사이드 이미지 처리 권장 (캡처 시에도 워터마크 유지)

## 06. 재열람 정책

| 구분 | 기본결과 · 상세리포트 | 오답확인 |
|------|----------------------|---------|
| 열람 방식 | 이메일 링크 → 이메일 인증 → 열람 | 결제 직후 웹 1회 열람 |
| 재열람 | ✅ 이메일 링크 통해 반복 열람 가능 | ❌ 불가 |
| 이메일 링크 | 자동 발송 | 발송 안 함 |
| 재접근 시도 | 이메일 인증 후 접근 허용 | 열람 완료 처리 후 재접근 차단 |

> ⚠️ **기획 이슈 — 세션 이탈 예외 처리**: 오답확인 1회 열람 중 세션 이탈(창 닫기, 네트워크 오류)로 실제 미확인 시 예외 처리 정책을 클라이언트와 사전 합의해야 합니다. 예시: 결제 완료 후 5분 이내에 한해 재접근 1회 허용, 이후 차단.

## 09. 모호하거나 리스크 있는 지점

**1. 오답확인 1회 열람 + 세션 이탈 충돌**
결제 직후 1회 열람인데, 브라우저 강제종료·네트워크 끊김 시 사용자는 "한 번도 못 봤다"고 주장할 수 있습니다. CS 분쟁 발생 가능성이 높으므로 예외 정책(타임아웃 재접근 허용 등)을 반드시 클라이언트와 확정해야 합니다.

**2. 이메일 인증 방식의 보안 수준**
현재 설계는 "응시 시 입력한 이메일 값 매칭"만으로 열람을 허용합니다. 이메일 주소를 아는 제3자가 접근할 수 있으므로, 보안 강화가 필요하면 OTP·수험번호 추가 입력 등을 고려해야 합니다.

**3. 기본결과 → 번들 업그레이드 차액 정책**
기본결과(14,900원) 구매 후 번들(28,900원)로 업그레이드 시 차액 14,000원 적용인지, 번들 전액 재결제인지 클라이언트 확정이 필요합니다. 차액 적용 시 부분환불 처리 로직이 결제 모듈에 필요할 수 있어 개발 공수에도 영향을 줍니다.

**4. 결과지 이메일 링크 유효기간**
이메일 발송 링크가 무기한 유효인지, 일정 기간 후 만료인지에 따라 DB 보관 정책과 사용자 안내 문구가 달라집니다. 클라이언트와 협의가 필요합니다.
