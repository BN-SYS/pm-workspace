## 2. 서비스/기능 구조

### 사용자 영역 (홈페이지)

| 영역 | 페이지 | 비고 |
|---|---|---|
| 홈 | 메인 | 히어로, 서비스 소개, CTA |
| 소개 | 브랜드/회사 소개 | (세부 메뉴 확인 필요) |
| 서비스 | 서비스 목록, 서비스 상세 | 예약 연결 CTA 포함 |
| 예약 | 예약 신청, 예약 확인 | 자체 예약시스템 연동 |
| 공지/커뮤니티 | 공지사항, FAQ | (운영 여부 확인 필요) |
| 마이페이지 | 예약 내역, 회원정보 수정 | 로그인 필수 |

### 관리자 영역

| 영역 | 기능 | 비고 |
|---|---|---|
| 예약 관리 | 예약 목록, 상태 변경, 상세 조회 | 핵심 |
| 스케줄 관리 | 가용 날짜/시간/정원 설정 | 핵심 |
| 고객 관리 | 회원 목록, 예약 이력 조회 | |
| 서비스 관리 | 서비스 등록/수정/삭제 | |
| 공지 관리 | 공지사항 등록/수정/삭제 | |
| 통계 | 예약 현황, 매출 요약 | (범위 확인 필요) |

### 예약시스템 핵심 플로우

```
사용자: 서비스 선택 → 날짜·시간 선택 → 정보 입력 → 결제(또는 문의) → 예약 완료
관리자: 예약 접수 확인 → 승인/거절 → 고객 알림 발송
```

## 3. 핵심 기능 명세

### 예약시스템

| 기능 | 설명 | 우선순위 |
|---|---|---|
| 서비스별 예약 | 서비스 단위로 날짜·시간·인원 선택 | 필수 |
| 실시간 가용 조회 | 잔여석/마감 여부 표시 | 필수 |
| 예약 신청 | 비회원/회원 모두 가능 여부 확인 필요 | 필수 |
| 예약 확인·취소 | 이메일·예약번호로 조회, 취소 정책 반영 | 필수 |
| 관리자 승인 방식 | 자동 확정 vs 관리자 승인 후 확정 | 협의 필요 |
| 알림 | 예약 완료·취소 이메일/SMS 발송 | 필수 |
| 결제 연동 | PG 연동 여부 (있을 경우 별도 스펙) | 협의 필요 |

### 회원 기능

| 기능 | 설명 |
|---|---|
| 회원가입/로그인 | 이메일 기반, 소셜 로그인 여부 확인 필요 |
| 마이페이지 | 예약 내역 조회, 취소 신청, 회원정보 수정 |
