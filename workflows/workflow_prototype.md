# Workflow: HTML 프로토타입 제작 (Figma 시안 없는 경우)

## 언제 쓰나
Figma 시안 없이 기능 요약만으로 프로토타입을 만들 때.
"프로토타입 만들어줘", "시연용 HTML 만들어줘" 라고 할 때.

---

## Step 1 — 전제 확인

- [ ] 기능 범위 확정 여부
- [ ] 클라이언트 업종/톤앤매너 (공공기관이면 보수적, 스타트업이면 모던)
- [ ] 반응형 필요 여부
- [ ] 시연 환경 (PC 브라우저 / 모바일 / 프로젝터)

---

## Step 2 — UI 방향 결정 (시니어 판단)

Figma 없을 때 기준:
- 색상: 클라이언트 기존 CI 색상 또는 업종 적합 색상 선택
- 폰트: Noto Sans KR (웹 안전 한글 폰트)
- 간격: 8px 그리드 기준
- 컴포넌트: Bootstrap 5 기반 또는 순수 CSS

---

## Step 3 — 구현 순서

```
1. HTML 단일 파일 구조 세팅
   - <head>: CSS 변수 정의, 외부 폰트/아이콘 CDN
   - <body>: 화면별 섹션 구분

2. CSS 변수 정의
   :root {
     --color-primary: #결정된색상;
     --color-bg: #f8f9fa;
     --font-body: 'Noto Sans KR', sans-serif;
     --spacing-unit: 8px;
     --border-radius: 6px;
   }

3. 공통 컴포넌트 먼저
   - 헤더/GNB
   - 사이드바 (있을 경우)
   - 푸터

4. 화면별 콘텐츠 구현
   - 화면 목록 순서대로
   - 더미 데이터로 채울 것 (실제처럼 보이게)

5. 인터랙션 추가
   - 탭, 모달, 드롭다운: Vanilla JS 또는 jQuery
   - 화면 전환: 섹션 show/hide 방식
```

---

## Step 4 — 품질 기준

- PHP 없이 HTML 단일 파일만으로 동작
- 더미 데이터는 실제처럼 (홍길동, 010-1234-5678 X → 맥락에 맞는 데이터)
- 주요 기능 구간에 개발팀용 주석 삽입
- 클릭 가능한 버튼은 모두 반응이 있어야 함 (비어있으면 안 됨)
- 빈 상태, 에러 상태도 구현

---

## Step 5 — 산출물 저장

- 저장 위치: `outputs/prototype_프로젝트명_v1.html`
- 이미지 리소스는 CDN 또는 Base64 인라인으로 처리 (외부 파일 의존 금지)

---

## Step 6 — PM 보고

- "N개 화면 구현했고, [화면명]은 더미로 처리했습니다"
- 클라이언트 시연 전 확인 포인트 목록 전달
