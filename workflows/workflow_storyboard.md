# Workflow: 화면설계서(스토리보드) 작성

## 언제 쓰나
기능정의서가 확정된 후, 화면 단위 설계가 필요할 때.
"스토리보드 써줘", "화면 목록 정리해줘", "화면 흐름 그려줘" 라고 할 때.

---

## Step 1 — 전제 확인

착수 전 확인:
- [ ] 기능정의서 확정 여부 (미확정이면 기능정의서부터)
- [ ] Figma 시안 유무 (있으면 시안 기반, 없으면 시니어 판단)
- [ ] 반응형 필요 여부 (PC only / PC+모바일 / 앱)
- [ ] 메뉴 구조(IA) 초안 유무

---

## Step 2 — 메뉴 구조(IA) 정의

```
GNB (Global Navigation)
├── 1뎁스 메뉴
│   ├── 2뎁스 메뉴
│   │   └── 3뎁스 메뉴 (필요시)
```

- 각 메뉴에 화면 ID 부여: `SCR-001`, `SCR-002` ...
- 권한별 접근 가능 메뉴 표시 (사용자/관리자 구분)

---

## Step 3 — 화면 목록 작성

| 화면 ID | 화면명 | URL 경로 | 접근 권한 | 설명 | 비고 |
|---|---|---|---|---|---|
| SCR-001 | 메인(홈) | / | 전체 | 서비스 진입점 | |
| SCR-002 | 로그인 | /login | 비로그인 | 이메일+비밀번호 | SNS 로그인 여부 확인 |

---

## Step 4 — 화면 상세 설계

각 화면별로:

```
[화면 ID] 화면명
- 목적: 이 화면이 하는 일 한 줄 요약
- 접근 경로: 어디서 오는지 (이전 화면)
- 주요 컴포넌트: 헤더 / 검색 / 리스트 / 버튼 등
- 액션 및 분기:
  - [버튼명] 클릭 → [다음 화면 ID] 이동
  - [조건] → [예외 화면] 처리
- 상태 케이스:
  - 기본 상태
  - 데이터 없음 (빈 상태)
  - 로딩 중
  - 에러 발생
- 유효성 검증: 필드별 조건 명시
- 관련 API/기능: 연결되는 기능 ID
```

---

## Step 5 — 산출물 생성

- 저장 위치: `outputs/화면설계서_프로젝트명_v1.0.docx`
- 화면 와이어프레임은 텍스트 기반 또는 ASCII로 표현 (Figma 없을 때)

---

## Step 6 — PM 보고

- "총 N개 화면, 주요 흐름은 이렇습니다"
- 확인 필요한 분기 케이스 목록 함께 전달
