# Workflow: 테스트 시나리오 작성

## 언제 쓰나
개발 완료 후 QA, 납품 전 검수가 필요할 때.

---

## Step 1 — 전제 확인

- [ ] 기능정의서 / 개발 스펙 확정본 확인
- [ ] 테스트 환경 (스테이징 서버 / 로컬)
- [ ] 테스트 담당자 (PM / 개발팀 / 클라이언트)
- [ ] 납기 일정

---

## Step 2 — 테스트 시나리오 구조

Excel 기준:

```
[Sheet 1: 테스트 케이스 목록]
| TC ID | 기능 대분류 | 기능명 | 테스트 항목 | 사전 조건 | 입력값 | 기대 결과 | 실제 결과 | 판정(Pass/Fail) | 비고 |

[Sheet 2: 결함 목록]
| 결함 ID | TC ID | 발견일 | 심각도 | 현상 | 재현 방법 | 담당자 | 수정일 | 상태 |
```

---

## Step 3 — 테스트 케이스 도출 기준

```
정상 케이스:
- 의도한 대로 사용했을 때

예외/경계 케이스:
- 빈 값 제출
- 최대값 초과 입력
- 특수문자 입력
- 권한 없는 사용자 접근
- 중복 데이터 처리

환경 케이스:
- 다른 브라우저 (Chrome, Edge, Safari)
- 모바일 뷰 (반응형 적용 시)
- 느린 네트워크 (로딩 상태 확인)
```

---

## Step 4 — 심각도 기준

| 심각도 | 기준 |
|---|---|
| Critical | 서비스 불가, 데이터 손실 |
| High | 핵심 기능 오동작 |
| Medium | 일부 기능 오동작, 우회 가능 |
| Low | UI 오류, 경미한 표시 문제 |

---

## Step 5 — 산출물 저장

- 저장 위치: `outputs/테스트시나리오_프로젝트명_v1.0.xlsx`

---

## Step 6 — PM 보고

- "총 N개 TC 중 Pass N / Fail N / 미테스트 N"
- Critical/High 결함은 즉시 개발팀 전달
- 납품 조건: Critical 0건, High 0건
