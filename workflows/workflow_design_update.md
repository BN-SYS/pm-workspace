# Workflow: 디자인 시안 반영 (기존 프로토타입 업데이트)

## 언제 쓰나
기존 프로토타입에 새 Figma 디자인 시안을 반영할 때.
"디자인 나왔어, 프로토타입 업데이트해줘" 라고 할 때.

---

## Step 1 — 현황 파악

착수 전 확인:
- [ ] 업데이트할 프로토타입 파일 경로 (`outputs/` 확인)
- [ ] 새 Figma 시안 수령 방식 (URL / 이미지)
- [ ] 전체 교체인지, 일부 화면만 변경인지

---

## Step 2 — 차이 목록 정리 (착수 전 PM 보고)

```
[기존 vs 시안 차이 목록]
1. 색상: Primary #기존 → #신규
2. 헤더 레이아웃: 1단 → 2단 구조 변경
3. 버튼 스타일: 둥근 모서리 → 직각
4. 폰트 사이즈: 본문 14px → 16px
5. 신규 컴포넌트: 태그 뱃지 추가
6. 시안에 없는 상태 (빈 상태, 에러) → 기존 유지 또는 새 스타일 적용 확인 필요
```

이 목록을 PM에게 먼저 보고 후 착수.

---

## Step 3 — 반영 순서

```
1. CSS 변수부터 교체 (색상, 폰트, 간격)
   → 개별 수정 절대 금지, 변수 일괄 교체 원칙

2. 공통 컴포넌트 교체 (버튼, 인풋, 카드 등)
   → HTML/CSS 블록 단위로 교체

3. 화면별 레이아웃 변경 적용

4. 이미지/아이콘 교체
   → CDN 또는 assets/figma/ 경로로 교체

5. 인터랙션/애니메이션 추가
   → CSS transition 우선, 복잡한 건 Vanilla JS
```

---

## Step 4 — 예외 처리 기준

| 상황 | 처리 방법 |
|---|---|
| 시안에 빈 상태/에러 없음 | 시니어 판단으로 새 스타일 적용, PM에 보고 |
| 시안과 기획 충돌 | 착수 전 PM 확인 필수. 임의 결정 금지 |
| 시안 미비 화면 | 기존 프로토타입 유지 + PM에 확인 요청 |

---

## Step 5 — 산출물 저장

- 버전 올리기: `v1.0` → `v1.1` 또는 `v2.0`
- 저장 위치: `outputs/prototype_프로젝트명_v1.1.html`
- 이전 버전 덮어쓰지 말 것 (버전 보존)

---

## Step 6 — PM 보고

- "이 4가지가 달랐고, 이렇게 반영했습니다"
- 시니어 판단으로 추가한 항목 명시
- 추가 확인이 필요한 항목 별도 표시
